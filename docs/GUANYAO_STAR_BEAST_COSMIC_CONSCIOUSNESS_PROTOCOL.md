# GUANYAO Star Beast Cosmic Consciousness Protocol

协议编号：`RC-STAR-BEAST-COSMIC-CONSCIOUSNESS-LAYER-P86`

P86 补齐艮之白虎 Genesis Prototype 的生命意识表达层。它不增加新的 Genesis 阶段，而是在 P85 五阶段中建立意识从休眠、形成、苏醒到静默在场的表达映射。

## 1. 数据链

`GenesisVisualState → CosmicConsciousnessState → Isolated Prototype Renderer`

`CosmicConsciousnessState` 只保存星尘流、星核呼吸、内部光流与 Presence 的表达引用，不复制生命事实、坐标、粒子数量、节奏数值或动画参数。

## 2. 生命意识不是装饰

`COSMIC_CONSCIOUSNESS` 不是装饰粒子，而是星兽内部正在运行的生命意识：

- 星尘意识流：极少量星尘沿生命结构稀疏、安静、持续地流动；禁止爆炸粒子。
- 星核呼吸：星核缓慢亮暗，不闪烁，不制造情绪刺激。
- 内部光流：艮之原力从星核向身体边界稳定沉积，不表现扩张或攻击。
- Presence：最终显化保持静默存在，不增加动作表演。

## 3. 五阶段意识映射

1. `COSMIC_FIELD`：意识休眠，生命内部尚未启动。
2. `WESTERN_MANSION_ALIGNMENT`：星辰形成秩序，意识仍悬停等待。
3. `WHITE_TIGER_FORMATION`：稀疏星尘开始沿星骨流动，星核出现微弱呼吸。
4. `GEN_INFUSION`：星核进入缓慢稳定呼吸，内部光流向边界沉积。
5. `WHITE_TIGER_REVEAL`：意识保持运行，形成安静、守望、在场的生命存在。

## 4. 工程边界

Mapping 只消费 `GenesisVisualState`，不得修改 P84 Asset Definition 或 Life State，不得创建 Memory、Growth、Crystal 或用户画像。Canvas 只消费 P85 Prototype Input 与 P86 表达状态，不得自行调用 Mapping。

Scope 始终固定为 `ISOLATED_PROTOTYPE_ONLY`。禁止接入 Launch、Gravity、Dynamics、Crystal、Storage 或正式用户流程；继续禁止 WebGL、Three.js、3D 模型、毛发、写实动物与战斗动作。
