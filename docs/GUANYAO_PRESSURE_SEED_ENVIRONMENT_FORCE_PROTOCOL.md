## 00｜协议定位

本协议依据《GUANYAO 世界观宪法 V2.0》，校准压力种子的产品定义、上卦边界、六维关系、前台语言与工程兼容范围。

本协议不是新的压力种子分类器，不改变 Pressure Seed Matrix、主修空间映射、PressureField、上卦形成、`currentHexagramProfile`、持久化或现有卡面。

本协议正式锁定：

> 压力种子 = 生命进入现实世界后遇到的环境力量（Environment Force）

压力种子不是一个人内部的缺陷，也不是系统对用户问题的诊断。它把一段正在影响生命的现实处境压缩成可被用户辨认、可被引擎分类、可进入六维体验的结构化入口。

## 01｜压力种子是什么

压力种子是：

> 一段现实环境力量在当前生命中的可辨认切片。

这里的“环境”包括：

- 权力关系与外部评价；
- 资源、利益与生存条件；
- 亲密关系中的不确定；
- 家庭责任与长期牵引；
- 群体位置、归属与社会期待；
- 存在价值、方向与未来的不确定。

压力种子帮助用户看见：

- 现在有什么现实力量正在作用于我；
- 它最先触动了我生命的哪个界面；
- 我正在怎样感受和回应它；
- 我是否仍有新的观察与选择空间。

它回答的不是“我哪里有问题”，而是：

> “此刻，什么现实力量正在经过我的生命？”

## 02｜压力种子不是什么

压力种子不是：

- 用户缺陷；
- 人格阴影证明；
- 心理诊断；
- 病理标签；
- 测试题；
- 系统审判；
- 失败原因；
- 用户应该被修复的部分；
- 母码的错误形态；
- 星兽受到的惩罚；
- 命运征兆；
- 上卦本身；
- 本局最终结果；
- Crystal 或资产生成按钮。

压力不是用户罪责。用户对压力产生反应，不等于用户软弱、错误或不成熟。

观爻不得通过放大焦虑、羞耻或危机感，让用户被迫认领一个压力种子。

## 03｜为什么叫“种子”

“种子”不是说问题源自用户内部，也不是说压力必然会长成灾难。

它表达三层含义：

1. 一段复杂现实被压缩成可以辨认的最小切片；
2. 这段现实力量进入体验后，可能触发身体、情绪、思想、行动、记忆与动机的变化；
3. 用户对它的觉察与选择，可能让这次经历留下新的生命印记。

压力种子不是缺陷种子，而是变化被看见的入口。

## 04｜正式结构：Surface / Core / Shell

现有三层结构保持不变：

### Surface｜用户辨认层

Surface 用一句具体、可感知的现实切片，让用户判断“这是否接近我正在经历的处境”。

Surface 不负责诊断，也不能替用户宣布事实。

### Core｜引擎分类层

Core 为引擎提供结构化信息，包括：

- `pressureField`；
- `pressureNature`；
- `relationshipRole`；
- `fieldBias`；
- semantic tags；
- six-dimension entry hints。

Core 不直接展示给用户，不得被包装成隐藏的心理结论。

### Shell｜体验影响层

Shell 帮助用户预感：如果这股环境力量继续作用，当前反应可能带来什么体验或代价。

Shell 不是威胁、恐吓或“你会失败”的预言。它必须保留可能性语言与用户解释权。

前台仍可只展示 Surface 与 Shell，Core 保持引擎内部消费。

## 05｜六类环境力量

现有六类 Pressure Field 保持：

- `POWER`：权力、控制与评价结构；
- `INTEREST`：资源、利益与交换冲突；
- `RELATION`：亲密关系与连接不确定；
- `FAMILY`：家庭责任与长期关系牵引；
- `SOCIAL`：群体位置、归属与社会评价；
- `EXISTENCE`：生存底盘、价值感与方向不确定。

它们描述环境力量的来源结构，不是六种用户类型。

`pressureNature` 描述力量如何作用，例如评价、资源冲突、依恋、控制、义务、归属、身份或生存压力；它不描述用户的人格性质。

`relationshipRole` 描述力量发生在哪一种关系中；它不把任何关系角色自动判定为好或坏。

## 06｜PressureSeed 与 PressureField

`PressureSeed` 与 `PressureField` 不能混为同一层：

### PressureSeed

描述：

> 什么现实环境力量正在进入这一局？

它承载具体处境、关系、性质与语义材料。

### PressureField

描述：

> 这股环境力量在当前窗口中作用得多强、多久、以怎样的重力范围发生？

它负责强度、风险窗口、`gravityValue` 与场域权重。

`PressureField` 不直接选择上卦，也不判断用户是否已经改变。

## 07｜压力种子分类与上卦

压力种子不是硬编码的一步上卦映射。

正式形成链保持：

```text
SelectedPressureSeedContext
→ 标准化 PressureSeed
→ Pressure Seed Semantic Classification
→ UpperCodeFormation
→ upperTrigram
```

上卦表达：

> 当前环境力量在这一局形成了怎样的外部处境结构？

它不是：

- 用户命运；
- 用户人格；
- 吉凶判断；
- 最终卦晶；
- 改变已经发生的证明。

压力种子分类只提供上卦形成语义。PressureField 只提供环境力量的强度与窗口。

## 08｜与母码和本局卦象的关系

冻结成卦链保持：

```text
母码 → 下卦

压力种子 → 压力种子分类 → 上卦

下卦 + 上卦 → currentHexagramProfile
```

三层语义分别是：

- 母码：生命携带的原始生命倾向；
- 压力上卦：现实环境力量形成的当前处境结构；
- `currentHexagramProfile`：原始倾向与环境力量相遇后形成的本局变化结构。

`currentHexagramProfile` 不是最终结果，不是人格标签，也不是 Crystal。

压力种子不能反向改写母码，母码也不能取消用户对当前现实处境的真实感受。

## 09｜与星兽的关系

星兽是生命本身，压力种子是生命在现实中遇到的环境力量。

压力种子不用于伤害、削弱或惩罚星兽。星兽也不负责替用户击败、吞噬或消灭压力。

更准确的关系是：

```text
星兽进入现实
→ 遇见环境力量
→ 在六维生命界面中产生体验
→ 在觉察中重新取得方向感
```

黑洞、扰动、重力等视觉隐喻可以继续存在，但不得表达：

- 用户让星兽受伤；
- 压力必须被彻底消灭；
- 星兽战胜敌人后用户才算成功；
- 未完成六维会导致生命能量亏损。

Gravity 是现实世界的牵引，不是用户失败的证据。

## 10｜与六维的关系

压力种子进入六维，不是为了分析用户有六种问题。

六维表达同一股环境力量如何通过生命的六个界面被体验：

- 身体：这股力量如何被身体感知；
- 情绪：它激起怎样的内在波动；
- 思想：生命如何解释正在发生的现实；
- 行动：生命如何停住、靠近、离开或回应；
- 记忆：它如何唤回过去的经验；
- 动机：它触动了什么真正想守住或寻找的方向。

Primary Petal / 主修空间只表示本局最适合先观察的入口，不表示其他维度没有发生，也不表示该维度损坏最严重。

Primary Petal 的选择不得以“哪里最羞耻、哪里最有问题”为原则，而应以“从哪里开始最清晰、最温和、最有选择空间”为原则。

## 11｜从压力到变化

观爻不是帮助用户消灭所有压力。

完整世界观链是：

```text
原始生命倾向
→ 遇到现实环境力量
→ 形成本局变化结构
→ 环境力量穿过六维生命体验
→ 用户看见一种正在发生的反应
→ Choice 重新取得生命方向
→ ChangeExperience 留下变化
→ Crystal 沉积为生命印记
```

压力种子只开启这一局，不保证变化已经发生，也不能单独生成 Crystal。

## 12｜前台语言

前台应该说：

- “这可能是你此刻正在经历的一股现实力量。”
- “先看看它正在怎样影响你。”
- “这不是你的错，也不需要一次解决全部。”
- “你可以从最有感觉的地方开始看见。”
- “压力仍可能存在，但你与它的关系可以发生变化。”

前台不应该说：

- “系统发现了你的问题。”
- “这就是你的心理缺陷。”
- “你的母码导致了这场压力。”
- “你必须完成六维才能消除黑洞。”
- “你没有战胜压力。”
- “你的能量不足。”
- “AI 已经识别出你真正的问题。”
- “这颗种子决定了你的结果。”

用户必须能够不认领、不继续或重新选择压力种子，而不受到惩罚性反馈。

## 13｜AI 与用户主体性

AI 可以帮助整理环境力量的语言和结构，但不能替用户宣布：

- 哪一段经历最重要；
- 用户真正害怕什么；
- 某个关系一定有害；
- 某种反应一定错误；
- 用户必须如何选择。

压力种子首先是一面现实镜面，不是 AI 对用户的隐藏评分。

AI 输出必须保留“可能、是否接近、你可以观察”的空间。最终是否认领一颗压力种子，由用户决定。

## 14｜工程兼容边界

本协议只校准产品语义，不重命名或修改：

- `GuanyaoPressureSeed`；
- `SelectedPressureSeedContext`；
- `StoredSelectedPressureSeedContext`；
- `PressureSeed`；
- `PressureField`；
- `PressureSeedMatrixNode`；
- `PrimaryPetalId`；
- `pressureSeedLanguageProtocol`；
- `currentHexagramProfile`；
- `UpperCodeFormation`；
- storage key 与 schema version。

`oldReactionHint`、`costHint`、`behaviorBlock` 等字段可以继续作为工程兼容输入，但前台必须通过 Presentation 层避免诊断、羞耻和缺陷语言。

## 15｜本刀不改变

本协议不改变：

- Pressure Seed Matrix；
- Surface / Core / Shell 数据结构；
- 六类 Pressure Field；
- 压力种子选择结果；
- Primary Petal 映射；

- `currentHexagramProfile`；

## 16｜协议优先级

压力种子语义优先级：

```text
GUANYAO 世界观宪法 V2.0
→ 星兽生命原型协议
→ 母码原始生命倾向协议
→ 压力种子环境力量协议
→ Pressure Seed Matrix / Causal Map / 页面表达
```

旧文档中的“人格行为系统、填补黑洞、化解压力、人格资产生成”等表述不得覆盖本协议。它们可作为历史工程语境保留，但后续前台施工必须逐刀校准。

## 17｜最终定义

压力种子不是系统发现的用户问题。

它是生命正在遇见的一股现实环境力量。

它不证明用户有缺陷，也不要求用户立即战胜现实。

观爻通过压力种子帮助用户看见：

> 此刻有什么力量正在经过我的生命，它正在怎样影响我，而我是否仍能重新选择与它相处的方向。
