# GUANYAO Genesis Birth Mansion Ignition Projection V1.0

状态：`ISOLATED_WEBGL_PROTOTYPE_ONLY`

## 目的

P104 不是点亮一颗星，而是让宇宙在时间回应之后，第一次认领一个生命种子。认领的是本命星宿的来源引用，不是直接展示星宿名称或制造新的身份。

## 数据边界

```text
Time Sequence Recognition Projection
+ Formal Mansion Result Reference
↓
Birth Mansion Ignition Projection
↓
Life Presence Projection
↓
Isolated WebGL Scene
```

P104 只消费 P103 的时间识别投影与正式星宿引擎结果引用。它不重新计算星宿，不复制星宿事实，不读取四象、MotherCode 或 Personal Star Beast。

## “宇宙认领”表达

- `seedClaimExpression`：表达生命种子被局部注意、锚定与认领的程度；
- `cosmicRecognitionExpression`：表达周围空间向种子收束、背景安静下来与方向回应；
- `ignitionStage`：`WAITING_FOR_TIME_RECOGNITION`、`SEED_APPROACHING`、`SEED_CLAIMED`，是体验阶段，不是新的计算结果。

这不是一个更亮的点。正确的感受是：宇宙场开始为一个生命种子让出位置。

## 严格边界

P104 禁止：

- 直接显示本命星宿名称；
- 四象显化；
- MotherCode 原力显化；
- Personal Star Beast Reveal；
- 动物轮廓、资产生产、正式 Runtime、UI 流程或 Storage 接入。

人工验收问题：用户是否感觉“这片宇宙第一次认领了我的生命种子”，而不是“某个星点被点亮了”。
