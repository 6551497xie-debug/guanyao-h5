# GUANYAO Genesis Full Sequence Review Protocol

`RC-GUANYAO-GENESIS-FULL-SEQUENCE-INTEGRATION-REVIEW-P8`

## 目的

本协议审查 P1–P7 Genesis 视觉语义是否形成一条连续的生命显化旅程，而不是七个彼此独立的视觉模块。P7.1「生命存在感」属于兽层的连续性增强，不新增第八个 Genesis 语义层。

审查只读取视觉状态之间已经建立的上游引用，不修改任何视觉状态，不改变生命系统、身份来源、Engine、Scene Model、RenderPlan、Renderer 或用户流程。

## 七层显化链

```text
月｜太阴入口
  ↓ 月华照见星河
星｜星河秩序
  ↓ 时间进入已有秩序
时｜生命时序共振
  ↓ 时间改变星河状态
象｜四象显影
  ↓ 运动规律沉积
卦｜变化印记
  ↓ 结构影响动力
力｜生命原力
  ↓ 生命逐渐归来
兽｜个人星兽存在
```

## 七层职责与验收

| 层 | 回答的问题 | 连续性要求 |
| --- | --- | --- |
| 月 | 生命进入什么宇宙 | 建立时间与空间入口 |
| 星 | 宇宙如何记录生命 | 月华自然照见星河 |
| 时 | 我的时间如何进入 | 像交付，而不是填写 |
| 象 | 生命如何呈现形态 | 来自星辰关系，不直接加载四象 |
| 卦 | 生命如何变化 | 变化沉积，不展示卦符结果 |
| 力 | 生命如何运行 | 结构影响动力，不增加技能效果 |
| 兽 | 生命如何存在 | 从前六层归来，形成安静的生命存在感 |

## 转场因果冻结

- **月 → 星**：月华照见星河；星空不能突然出现。
- **星 → 时**：时间进入已有秩序；输入不能跳过响应直接显示结果。
- **时 → 象**：时间改变星河状态；不能直接加载四象。
- **象 → 卦**：形态运动留下变化轨迹；不能突然弹出卦符。
- **卦 → 力**：变化结构影响动力方式；不能新增能力效果。
- **力 → 兽**：生命逐渐归来；不能使用召唤动画或英雄登场。

## 边界

本审查属于 `Genesis Integration`，只输出 `GenesisFullSequenceReview`。它不计算身份、不读取用户数据、不读取 Engine Result、不创建 Renderer Command，也不接入 Reality、Gravity、Choice、Crystal 或 Storage。

视觉验收应保持：慢、静、东方、连续、有生命感；避免游戏生成、神兽召唤、测试报告感与动物化路径。
