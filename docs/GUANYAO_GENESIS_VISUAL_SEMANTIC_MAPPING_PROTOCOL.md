# RC-GUANYAO-GENESIS-VISUAL-SEMANTIC-MAPPING-PHASE-P0

# GUANYAO Genesis Visual Semantic Mapping V1.0

本协议把已经冻结的 Genesis 产品语义转换为可施工的视觉表达规则。视觉不创造生命，视觉只显化生命。

## 七层映射

```text
月｜太阴入口
↓
星｜星河秩序
↓
时｜生命时序共振
↓
象｜四象显影
↓
卦｜变化印记
↓
力｜生命原力
↓
兽｜个人星兽归来
```

### 月｜太阴入口

表达时间入口：深空、月华、圆月、太阴光场、慢节律。此层不显示生命结果、星兽暗示或身份提示，目标是让用户进入天地时间场。

### 星｜星河秩序

表达天地星辰坐标：星河、七宿关系、四象七宿流转、星辰结构。此层不直接出现动物，也不把星辰做成星座图鉴，目标是让用户感受到星辰有序。

### 时｜生命时序共振

表达生命降临时间进入宇宙：月相变化、节律变化、星河响应、时间归位。此层不直接显示计算结果，不提前显示个人星兽，目标是让用户感到时间进入宇宙。

### 象｜四象显影

四象是星辰秩序聚合后的生命形态场。允许形态场、空间变化、七宿聚合、四象骨架；禁止青龙、白虎、朱雀、玄武模型。象由星辰关系自然形成，不由视觉选择动物。

### 卦｜变化印记

表达生命变化规律：阴阳节律、六层结构、变化轨迹、印记形成。禁止卦卡展示与命理解释页，目标是让用户感受到变化有序。

### 力｜生命原力

表达 MotherCode / LifeArchetype 原力：节律变化、运动方式变化、核心状态变化、生命呼吸变化。禁止属性标签、能力数值和技能效果，目标是让力量苏醒。

### 兽｜个人星兽归来

个人星兽必须由星宿结构、四象形态场与原力表达共同形成。禁止直接生成神兽图片、宠物化、游戏角色化或动物身份硬编码，目标是让用户看见自己的生命形态。

## 施工五问

每次 Genesis 视觉施工必须回答：

1. 属于月／星／时／象／卦／力／兽哪一层？
2. 表达什么语义？
3. 是否改变生命来源？
4. 是否只是视觉表达？
5. 是否需要跨层评审？

本层属于视觉治理层，不是 Identity Layer。不得反向修改 Life System、Identity Source、Engine、Scene Model 或 RenderPlan。
