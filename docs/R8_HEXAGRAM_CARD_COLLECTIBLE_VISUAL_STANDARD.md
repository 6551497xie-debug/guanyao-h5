# R8｜卦码卡收藏级视觉标准锁定文档 V1

## 一、文档定位

本文件用于锁定《观爻 / GUANYAO SANDBOX》64 张卦码卡的收藏级视觉标准。

本标准不是生图提示词。
本标准不是模板开发文档。
本标准不是单张卦卡说明书。

本标准用于回答：

1. 什么样的图只能作为占位图？
2. 什么样的图可以作为当前入仓测试图？
3. 什么样的图才可以成为收藏级卦码卡？
4. 每张卡必须具备什么唯一视觉符号？
5. A 面无字底图如何为金句、标签、LOGO 留出结构空间？
6. 如何避免 64 张都变成黑金 AI 科幻废墟图？
7. 如何保持系列统一，同时让每张卡有不同骨相？

本文件从 R8 开始作为后续 64 张卦码卡视觉验收、重制、入仓、发布的统一标准。

4. /hexagram-card-lab 预览验证

3. 可分享传播图

不得直接等同于：

正式发行收藏版 / 文明母卡最终版

## 三、收藏级卦码卡的基本定义

一张收藏级卦码卡，必须同时满足四个条件：

### 1. 它是观爻的

第一眼看上去，它必须属于《观爻 / GUANYAO SANDBOX》的视觉宇宙。

它可以黑，可以金，可以深，可以冷，但不能只是普通 AI 黑金科幻图。

它必须具备：

* 黑金文明感
* 抽象结构感
* 人格处境感
* 行为命运感
* 东方卦码秩序感
* 非普通海报感
* 非游戏场景设定感

### 2. 它是这一卦的

每一张必须有不可替代的视觉母题。

不能出现：

* 换个标题就能当别的卦
* 换个编号就能继续用
* 只是黑金氛围不同
* 只是光柱位置不同
* 只是裂缝形状不同

收藏级卡面必须让用户感觉：

“这一张只能是这一卦。”

### 3. 它能承载 A 面模板

无字底图不是独立海报。
它必须服务 A 面结构。

它需要预留：

* 顶部卦名 / 卡名区
* 主视觉识别区
* 中下部金句区
* 标签区
* 底部品牌区

不能出现：

* 主视觉铺满整张，叠字后很乱
* 亮点正好压住金句
* 底部过亮导致标签不可读
* 主体太靠下导致品牌区拥挤
* 画面太碎导致卡面廉价

### 4. 它能被收藏和分享

收藏级卡面必须具备“拿出去给别人看”的完整感。

它应具有：

* 识别度
* 记忆点
* 仪式感
* 克制感
* 系列感
* 单张完整性
* 分享时的情绪钩子

不能只是：

* 当前项目能跑
* 图像没文字
* 看起来还行
* 大概符合 prompt
* 能填进模板

## 四、四级视觉分级标准

后续所有卦码图统一按 A / B / C / D 分级。

### A 级：收藏级 / 可发布级

可进入正式收藏版。

标准：

1. 用户主观认可。
2. 第一眼能识别该卦母题。
3. 有明确唯一视觉符号。
4. 不像普通 AI 场景图。
5. 不像其它卦的变体。
6. 无字底图为 A 面叠字留出安全区。
7. 黑金体系统一但骨相独立。
8. 可作为后续同拓扑卡的参考母版。
9. 可分享，可收藏，可用于正式发布。

A 级才可以标记为：

collectible-ready

### B 级：当前入仓 / 模板验证级

可进入项目正式资产槽，用于当前版本跑通。

标准：

1. 用户可以接受。
2. 无文字、无 LOGO、无卦符。
3. 主视觉大体符合本卦方向。
4. 能在 /hexagram-card-lab 中正常展示。
5. 不严重污染模板。
6. 但仍可能存在 AI 感、同质化、叠字空间不足等问题。

B 级只能标记为：

v0.8-usable

不能标记为最终收藏级。

### C 级：方向候选 / 不入正式槽

可保留在 workbench 或 candidates。

标准：

1. 有部分方向价值。
2. 某些构图或气质可参考。
3. 但整体不适合正式入仓。
4. 可能存在风景化、祭坛化、光柱化、概念图化。
5. 可作为下一轮提示或修正参考。

C 级只能作为：

candidate-reference

### D 级：废弃

不得进入正式资产槽，不得作为参考。

标准：

1. 用户明确不满意。
2. 违反 A 面规则。
3. 出现文字、LOGO、卦符、边框。
4. 完全套用错误骨相。
5. 严重风景化 / 人物化 / 祭坛化 / UI 化。
6. 与观爻宇宙不一致。
7. 容易误导后续生成。

D 级必须标记为：

rejected

## 五、无字底图硬性规范

所有正式无字底图必须满足：

1. 无文字
2. 无 LOGO
3. 无编号
4. 无卦名
5. 无卡名
6. 无金句
7. 无标签
8. 无边框
9. 无卦符
10. 无人物
11. 无 UI 元素
12. 无水印
13. 3:4 竖版
14. 黑场充足
15. 主视觉明确
16. 能适配 A 面模板叠字

特别锁定：

卦符只允许出现在 B 面顶部信息栏。
A 面不得出现卦符。
无字底图更不得出现卦符。

## 六、A 面叠字安全区标准

每张无字底图必须预留结构空间。

### 顶部 15%

用途：

* NO.编号
* 卦名
* 卡名

要求：

* 不要有强光爆点
* 不要有复杂碎片
* 不要有高对比纹理
* 不要让主视觉主体顶到标题区域

### 中部 45%

用途：

* 主视觉核心

要求：

* 这里是卡片第一识别区
* 必须放置本卦唯一视觉符号
* 主视觉不能散
* 不能所有卡都靠中轴光柱撑场

### 中下部 25%

用途：

* 金句
* 情绪钩子

要求：

* 必须有相对暗场
* 不能有复杂细节
* 不能有高亮光源压字
* 文字叠上去后要清晰

### 底部 15%

用途：

* 三标签
* GUANYAO SANDBOX / 观爻

要求：

* 底部不宜太亮
* 不宜堆满主体
* 不宜有强纹理干扰标签
* 必须有呼吸感

## 七、系列统一规则

64 张卦码卡必须统一：

1. 黑金基调
2. 抽象结构感
3. 深黑宇宙 / 文明切片气质
4. 非人物
5. 非自然风景
6. 非 UI 页面
7. 非普通塔罗
8. 非玄学符号堆砌
9. A/B 面固定模板
10. B 面固定卦形库调用
11. 标签无外框
12. 金句左对齐
13. LOGO 小而克制

---

## 八、系列差异化规则

64 张卦码卡必须差异化：

1. 每张有自己的主结构
2. 每张有自己的压力动作
3. 每张有自己的空间骨相
4. 每张不能仅靠色温变化区分
5. 每张不能都用中轴光柱
6. 每张不能都用裂缝
7. 每张不能都用黑色方块城市
8. 每张不能都用雾
9. 每张不能都用圆盘平台
10. 每张不能都用悬浮碎块

每一张都必须回答：

“如果去掉卦名和文案，这张图还能不能被认出是哪一种处境？”

如果不能，则不达收藏级。

## 九、六类首批拓扑收藏级要求

### 1. 001｜乾为天｜高空

收藏级识别符号：

* 高位
* 孤立
* 承压
* 向上
* 分力未发生

风险：

* 祭坛化
* 光柱化
* 平台化
* 神圣启动感

收藏级要求：

不是“光柱打在平台上”，而是“高位结构正在独自承压”。

---

### 2. 002｜坤为地｜大地

收藏级识别符号：

* 承载
* 厚重
* 下沉
* 地层
* 边界承受
* 吞下过多压力

风险：

* 风景化
* 地貌化
* 地球化
* 坤版祭坛

收藏级要求：

不是“漂亮大地”，而是“大面积承载后的边界压力”。

---

### 3. 003｜水雷屯｜破土

收藏级识别符号：

* 第一道缝
* 顶开旧土
* 初始阻力
* 破开但尚未完全展开

风险：

* 灾变裂谷
* 峡谷风景
* 地裂奇观
* 蓝光特效

收藏级要求：

不是“大地裂开”，而是“第一步顶开旧土”。

---

### 4. 004｜山水蒙｜迷雾

收藏级识别符号：

* 遮蔽
* 未明
* 问题未显形
* 轮廓被雾包住
* 看不清真正的问题

风险：

* 仙境云雾
* 山水风景
* 城市雾景
* 雾化 003

收藏级要求：

不是“雾很美”，而是“问题被雾遮住”。

---

### 5. 005｜水天需｜等待

收藏级识别符号：

* 悬置
* 未落
* 等待对齐
* 时机未开
* 结构未闭合
* 准备但不贸然行动

风险：

* 海景
* 湖面
* 日出
* 浪漫等待
* 水面风景
* 水上的 001

收藏级要求：

不是“等日出”，而是“结构尚未落位，时机还没打开”。

---

### 6. 006｜天水讼｜争鸣

收藏级识别符号：

* 左右对峙
* 立场分开
* 中央边界显现
* 不能再被吞掉
* 无法继续假装合在一起

风险：

* 光柱劈城
* 祭坛反应堆
* 灾变现场
* 战争场面
* 城市峡谷

收藏级要求：

不是“爆炸冲突”，而是“边界被迫显现”。

## 十、当前 001—006 临时分级建议

以下为当前阶段建议分级，用于内部判断，不等同于最终发布结论。

### 001｜乾为天｜高空

当前等级：

B / V0.8 可用

说明：

可用于当前项目入仓和模板验证，但仍有中轴、光柱、祭坛感风险。后续收藏级版本需进一步去祭坛化，强化高位独自承压。

### 002｜坤为地｜大地

当前等级：

B / V0.8 可用

说明：

方向可用，可入仓验证。后续收藏级版本需继续强化地层承载与边界压力，避免成为普通地貌图。

### 003｜水雷屯｜破土

当前等级：

B+ / 接近收藏级拓扑母版

说明：

已有确认图较强，裂缝识别明确。后续收藏级版本可在保留近景裂缝结构的基础上，微调色彩统一与 A 面叠字安全区。

### 004｜山水蒙｜迷雾

当前等级：

B / V0.8 可用

说明：

方向可用，但需警惕城市雾景化。收藏级版本应更强调“问题被遮住”的视觉符号，而不是简单迷雾空间。

### 005｜水天需｜等待

当前等级：

B / V0.8 可用候选

说明：

几何悬置方向优于水面风景方向。收藏级版本需进一步强化“结构未落位、时机未开”，避免成为黑金深井场景。

### 006｜天水讼｜争鸣

当前等级：

B / V0.8 可用候选

说明：

左右对峙方向成立，但后续需继续降低光柱感和 AI 黑城感，强化“边界被迫显现”的符号性。

## 十一、后续工作原则

从本文件入库后，后续卦码卡生产必须遵守以下原则：

1. 不继续盲目推进 007。
2. 先用本标准评估 001—006 当前图。
3. 当前图可以入仓作为 V0.8，但不得直接称为收藏级。
4. 收藏级版本后续单独重制。
5. 每张新图必须先判断：
    * 是否符合该卦母题？
    * 是否有唯一识别符号？
    * 是否为 A 面留出叠字空间？
    * 是否像观爻文明卡，而不是普通 AI 场景？
6. Codex 只负责工程接入，不负责审美判断。
7. 用户确认“这张可以”之前，不得入正式收藏级资产。
8. 任何被用户否定的图，不得作为 reference 继续污染后续生成。

## 十二、当前结论

当前 001—006 可继续作为 R8 V0.8 入仓测试版推进，但不能被误判为最终收藏级。

下一阶段应区分两条线：

### 工程线

继续把已确认的 V0.8 图导入项目，跑通模板、A/B 面、路径和 build。

### 收藏级线

另起一轮，按照本标准对 001—006 逐张重制或精修，直到达到可收藏、可分享、可发布标准。

最终锁定：

入仓不等于收藏级。
可用不等于可发布。
黑金不等于观爻。
好看不等于这一卦。
无字底图必须为 A 面服务。
64 张必须形成系列，也必须各自不可替代。
