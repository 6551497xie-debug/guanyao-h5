# R8｜卦码卡多模板参考体系锁定文档

## 1. 为什么不能只用 001 作为模板

001｜乾为天｜高空 的视觉特征过强：

* 中轴
* 上升
* 高位
* 光源集中
* 几何基座
* 开局承压

如果把 001 当作全局模板，后续卦卡会被误导成：

* 祭坛化
* 光柱化
* 平台化
* 装置化
* 所有卡都变成同一种中轴结构

因此 001 只能作为局部拓扑参考，不是全局模板。

核心锁定：

001 只能作为“乾类 / 上升型 / 高位承压型”的拓扑参考，不能作为 64 张卦码卡的全局视觉模板。

## 2. 三层参考体系

必须锁定三层：

### 第一层：固定模板层

负责统一：

* A/B 面比例
* 字体层级
* 卦名 / 卡名排版
* 金句位置
* 三标签样式
* LOGO 位置
* A 面不得出现卦符
* B 面固定完整六爻卦形
* 边框与留白

这一层由固定 React 模板控制，不由图片生成控制。

### 第二层：全局风格层

负责统一：

* 深黑底
* 冷金 / 米金
* 抽象结构
* 非实景化
* 非人物化
* 非风景化
* 负空间
* 克制光源
* 非满屏粒子
* 非装置化

这一层需要 2—4 张全局风格参考，但不得让任意一张成为全部卡的构图模板。

### 第三层：拓扑分流层

负责决定每张卡的骨相。

不同卦卡必须按拓扑分流，不得全部套用 001。

## 5. 后续生产规则

后续每张卡必须同时使用四个输入：

1. 固定 A/B 面模板
2. 全局风格参考
3. 拓扑分流参考
4. 本卦专属 visual brief

禁止使用：

* 单张 001 推导全部卦卡
* 单一光柱结构套所有卡
* 单一祭坛基座套所有卡
* 只看 prompt 不看拓扑
* 候选图未验收就入正式资产槽
