# GUANYAO Genesis Symbol Visual Calibration Protocol

RC-GUANYAO-GENESIS-SYMBOL-VISUAL-CALIBRATION-P4
Genesis 第四层：象｜四象显影

## 1. 目的与边界

本协议把 Genesis 第四层「象｜四象显影」落为视觉状态与映射契约，只消费 P3 Time Resonance Visual State，不改变生命计算、星宿算法、正式四象结果或产品流程。

视觉中的象不是计算中的四象结果，也不是四象动物资产。四象显影是星辰关系形成的生命形态场：用户感受到“星辰正在形成一种生命秩序”。

固定上游与下游：

```text
时间进入
↓
星河响应
↓
七宿聚合
↓
象形成
↓
卦（后续层，当前不进入）
```

本层只建立象稳定的视觉语义，不继续聚合成完整生命。

## 2. Symbol Visual State

`GenesisSymbolVisualState` 表达：

- 形态场存在；
- 七宿关系增强与聚合趋势；
- 星辰骨架形成；
- 四种抽象生命运动方式共存；
- 象在慢节律中稳定。

它只保存视觉表达引用与边界，不包含 Identity、Birth Mansion Result、Four Symbol Result、Hexagram、MotherCode 或 Personal Star Beast。

## 3. 七宿聚合

七个星辰节点的关系增强，星轨与星骨结构出现，聚合趋势变强。禁止七宿图鉴、宿名展示与动物轮廓。

目标是让用户感觉：星辰关系正在组织出生命秩序，而不是观看知识图表。

## 4. 四象形态场

四象不是出现，也不是青龙、白虎、朱雀、玄武模型。它们在本层只作为抽象的生命运动规律：

| 抽象运动 | 视觉语义 |
| --- | --- |
| 生发、延展、连接 | 向外展开、曲线延伸、星流环绕 |
| 收束、凝聚、边界 | 内聚、稳定、结构收紧 |
| 升腾、表达、显化 | 向上运动、光流展开、节律增强 |
| 承载、包裹、沉潜 | 环绕、内层保护、深度增强 |

这些是视觉形态场，不是用户身份，也不是正式四象计算结果。

## 5. 时间到象

时间进入后先由星河回应，再发生七宿聚合，最后形成象。时间输入不得直接生成四象，象必须来自星辰运动后的自然结果。

本层完成“象稳定”，不进入卦、力、兽：

- 不提前生成卦；
- 不提前生成 MotherCode；
- 不提前生成 Personal Star Beast；
- 不提前生成动物身份。

## 6. 工程边界

本刀不修改月层；不修改星层；不修改时层。

Symbol Mapping Service 只负责：

```text
Time Resonance Visual State
↓
Symbol Visual State
```

它不调用 Engine、Identity Source、Scene Model、RenderPlan 或 Renderer，不修改月、星、时层，不写 UI、Storage，也不生产资产。

4. 是否没有提前出现星兽？
