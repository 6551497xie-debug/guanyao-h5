# GUANYAO 36 Yao Model Sample Protocol

## 00. Protocol Position

This document tests whether the Guanyao model of:

```text
Persona space x Yao stage
```

can become a real persona behavior dynamics unit.

This is a complete six-space / six-yao sample model.

It is still not a production data table.

This document defines six sample spaces:

- Body Space x 6 Yao
- Emotion Space x 6 Yao
- Thought Space x 6 Yao
- Action Space x 6 Yao
- Memory Space x 6 Yao
- Motivation Space x 6 Yao

The goal is to validate whether the model can drive:

- user language;
- current-session revision action;
- starbeast change;
- future engine mapping.

## 01. 36-Yao Model Principle

36 yao are not:

- 36 lines of copy;
- 36 tips;
- 36 tasks;
- 36 progress labels;
- 36 personality conclusions.

36 yao are:

> 36 observation units of how a persona model changes under real pressure.

Each unit must include:

1. Persona space.
2. Yao stage.
3. Pressure entry mode.
4. Old persona model.
5. Inertia reaction.
6. User-facing seeing language.
7. Anti-instinct direction.
8. Current-session revision action.
9. Starbeast change.

The unit is only valid if it can answer:

- What is happening to the user now?
- What old model is being exposed?
- What can the user see without shame?
- What small new response can happen in this round?
- What would the starbeast show if this model moved?

## 02. Body Space Sample

Core question:

> How does pressure enter the body?

The body space observes the user's first bodily registration of pressure: tension, breath change, heaviness, contraction, alertness, fatigue, or numbness.

### Body x First Yao: Trigger

Pressure entry mode:

Pressure first enters through bodily sensation.

Old persona model:

The body reads pressure as danger before the mind has finished understanding it.

Inertia reaction:

Immediate tightening.

User-facing seeing language:

Your body felt the pressure earlier than you did.

Anti-instinct direction:

Feel first. Do not rush to react.

Current-session revision action:

Pause and notice what is happening in the body.

Starbeast change:

The beast body begins to contract; the surrounding light field becomes disturbed.

### Body x Second Yao: Takeover

Pressure entry mode:

The pressure begins to control bodily state.

Old persona model:

Staying alert is the only way to stay safe.

Inertia reaction:

Continuous tightening.

User-facing seeing language:

You have learned to keep yourself ready for impact.

Anti-instinct direction:

Allow the body to lower one layer of defense.

Current-session revision action:

Release one point of tension you are still holding.

Starbeast change:

The skeletal light track tightens and pulls inward.

### Body x Third Yao: Explanation

Pressure entry mode:

The bodily reaction begins to influence interpretation.

Old persona model:

If the body feels danger, the situation must be dangerous.

Inertia reaction:

Risk is amplified.

User-facing seeing language:

Your body reaction is starting to judge reality for you.

Anti-instinct direction:

Separate sensation from fact.

Current-session revision action:

Name one fact that has actually happened.

Starbeast change:

The light track forks; one branch follows sensation, the other searches for fact.

### Body x Fourth Yao: Solidification

Pressure entry mode:

The bodily reaction becomes a default state.

Old persona model:

Long-term tension becomes the normal way to exist.

Inertia reaction:

The abnormal state is no longer noticed.

User-facing seeing language:

You have treated this state as normal for too long.

Anti-instinct direction:

Sense yourself again before continuing.

Current-session revision action:

Identify one body pattern that keeps repeating.

Starbeast change:

Old light tracks loop around the beast body.

### Body x Fifth Yao: Seeing

Pressure entry mode:

The user begins to see the body model.

Old persona model:

The body has been protecting automatically.

Inertia reaction:

The user keeps following the old protection route.

User-facing seeing language:

You are starting to see that this is not the only way to respond.

Anti-instinct direction:

Observe the reaction without being controlled by it.

Current-session revision action:

Give the body one new response before acting.

Starbeast change:

Star dust gathers back toward the body center.

### Body x Upper Yao: Revision

Pressure entry mode:

The body forms a new possible response.

Old persona model:

Tension protects.

Inertia reaction:

The old route tries to tighten again.

User-facing seeing language:

Your body is beginning to remember another kind of safety.

Anti-instinct direction:

Stabilize first, then act.

Current-session revision action:

Allow yourself to become steady before taking the next step.

Starbeast change:

The beast body returns to stable bearing; the core light grows stronger.

## 08. Validation Standard

The six sample spaces are valid if they are more useful than generic persona analysis.

They must be able to drive:

- user language that feels recognizable;
- current-session revision actions that are small and executable;
- starbeast changes that map to the user's model movement;
- future engine mapping without turning into a task system.

They must not become:

- personality diagnosis;
- coaching advice;
- inspirational copy;
- behavior training;
- check-in;
- score;
- 384 yao direction.

## 09. Sample Validation

### A. Does Body x 6 Yao Work?

Yes.

The body sample is close to user experience because many real pressures first appear as sensation before explanation.

It can drive:

- user language: "Your body felt the pressure earlier than you did."
- revision action: "Pause and notice what is happening in the body."
- starbeast change: contraction, bearing, core light.

Risk:

Some phrases can become too therapeutic if expanded too much. The production layer must keep them short.

### B. Does Action x 6 Yao Work?

Yes.

The action sample is close to user experience because many old models become visible only when the user notices what they are about to do.

It can drive:

- user language: "You are used to using action to erase uncertainty."
- revision action: "Name what is still unclear before acting."
- starbeast change: direction line, forward claws, old path loop, new path.

Risk:

Some phrases can become coaching language. The frontend must avoid "should" and avoid turning the revision action into a task.

### C. Do Emotion, Thought, Memory, And Motivation Work?

Yes.

Emotion works because it captures the moment when feeling begins to judge reality.

Thought works because it captures the moment when explanation becomes protection.

Memory works because it captures the moment when old traces cover current evidence.

Motivation works because it captures the hidden thing the user is trying to protect.

Each space can drive:

- user language that feels recognizable;
- a small current-session revision action;
- a starbeast change that reflects the model movement.

Risk:

Emotion can become therapy language, thought can become analysis language, memory can become trauma language, and motivation can become life-advice language. Production copy must stay concrete and restrained.

### D. What Still Feels Like AI Language?

The following kinds of expressions should be watched:

- "old persona model";
- "behavioral inertia";
- "anti-instinct direction";
- "separate action from self-worth";
- "current-session revision action".

These are protocol-level phrases.

They should be translated before production frontend copy.

### E. Do The Six Spaces Form One Model?

Yes.

The six spaces now form one model:

- Body: pressure enters sensation.
- Emotion: pressure enters feeling.
- Thought: pressure enters interpretation.
- Action: pressure becomes behavior.
- Memory: pressure activates old traces.
- Motivation: pressure exposes what the user protects.

The model should not be read as 36 frontend items.

Each unit must preserve:

```text
pressure entry
-> old model
-> inertia
-> seeing language
-> revision direction
-> starbeast change
```

## 10. Future Product Position

36 units should be a core backend/product model.

They should not be a full frontend display model.

Frontend should show only the part needed in the moment:

- current space;
- current yao stage;
- one seeing line;
- one revision direction;
- one starbeast cue.

The full 36-unit table should remain behind the experience as the semantic and behavioral model.

## 11. Boundary

This sample protocol does not:

- modify code;
- create 36 production data entries;
- modify `GravityPage.tsx`;
- add storage;
- add runtime;
- add routes;
- connect 384 yao direction;
- connect yao-device;
- connect archive;
- connect old R8.
