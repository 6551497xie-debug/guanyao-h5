## 00｜协议定位

本协议依据《GUANYAO 世界观宪法 V2.0》，校准母码的产品定义、计算边界、星兽关系、前台语言与工程兼容边界。

本协议不是新的母码引擎，不改变农历转换、地支时数、卦以八除、八卦落点、MotherCode Registry、持久化或现有卡面。

本协议正式锁定：

> 母码 = 原始生命倾向（Original Life Tendency）

母码不是系统对用户作出的人格结论，而是一个生命来到现实世界时，最自然地感受、启动、承载、照见、进入、穿越、停住或连接世界的初始倾向。

## 01｜母码是什么

母码是：

> 星兽来到现实世界时携带的原始生命倾向。

这是一条世界观关系，不是计算依赖关系。

“母”表达源头、初始与生成位置，不表达性别、家庭角色或外部权威。“码”表达一套可重复的符号落点，不表示人的生命可以被编码穷尽。

母码帮助用户看见：

- 我最自然地如何进入世界；
- 我遇到变化时首先调用什么生命力量；
- 同一种力量在压力下可能如何收紧或偏转；
- 我如何在觉察后更自由地使用这股力量。

母码回答的不是“我是谁”，而是：

> “我带着怎样的原始倾向来到这里？”

## 02｜母码不是什么

母码不是：

- 人格标签；
- 性格测试结果；
- 心理诊断；
- 优缺点评分；
- 能力等级；
- 命运结论；
- 终身不变的行为脚本；
- AI 生成的人设；
- 星兽四象归属的另一种写法；
- 64 卦本局卦象；
- Crystal 或最终生命印记。

母码不告诉用户“你只能这样”。它只让一种原本难以觉察的生命起势方式变得可见。

原始倾向也不等于天生优势。每一种倾向都可能在不同环境中展开、收紧、过度使用或形成新的表达。

## 03｜八母码，不是六十四母码

观爻当前母码正式范围是八个先天八卦落点：

- 乾：启动、创造与定向；
- 兑：连接、交换与转化；
- 离：照见、表达与辨明；
- 震：行动、启动与破开停滞；
- 巽：进入、感知缝隙与顺势展开；
- 坎：深入、承压与穿越困局；
- 艮：停住、建立边界与保留空间；
- 坤：承载、接住与稳定关系。

这些词是生命倾向语言，不是八种人格分类。

历史文档中的“观爻母码｜64”属于旧三层码制语境，不得覆盖当前冻结结构：

```text
8 母码 = 原始生命倾向
64 卦象 = 本局变化结构
Crystal = 经历之后留下的生命印记
```

母码不因进入 64 卦结构而变成 64 种母码。

## 04｜母码正式生成链

母码生成链保持不变：

```text
用户输入公历出生年、月、日与出生时辰
→ 公历出生日期转换为中国农历年、月、日
→ 出生时辰的地支转换为序数
→ 农历年数 + 月数 + 日数 + 地支时数
→ 卦以八除
→ 先天八卦落点
→ 映射到 8 母码 Registry
→ MotherCodeProfile
```

当前地支时数保持：

```text
子=1 / 丑=2 / 寅=3 / 卯=4 / 辰=5 / 巳=6
午=7 / 未=8 / 申=9 / 酉=10 / 戌=11 / 亥=12
```

计算公式保持：

```text
fieldSeed = lunarYear + lunarMonth + lunarDay + hourBranchOrdinal
fieldRemainder = fieldSeed mod 8
```

余数映射保持：

```text
1=乾 / 2=兑 / 3=离 / 4=震
5=巽 / 6=坎 / 7=艮 / 0或8=坤
```

地支序数是传统时序协议中的位置数，用于表达这一时段参与母码落点的时间属性。它不是现代科学意义上的能量测量值，也不构成宿命因果证明。

## 05｜输入与排除边界

母码正式输入只包括：

- 公历出生年；
- 公历出生月；
- 公历出生日；
- 出生地支时辰。

母码生成不使用：

- 出生地点；
- `geo` 方位；
- 星兽 `fourSymbol`；
- 星兽 `symbolicTrigram`；
- 压力种子；
- 用户当前情绪或行为；
- AI 推断；
- 当前本局卦象；
- Crystal 结果。

同一套历法协议下，相同出生年月日与地支时辰必须得到相同母码。

母码结果必须保持：

```text
trigramIndependentOfStarbeast = true
trigramIndependentOfLocation = true
trigramSourceScope = LUNAR_CHRONO_ONLY
```

## 06｜母码与星兽

星兽与母码来自两条独立链：

```text
出生日期
→ 农历月日
→ 二十八宿
→ 四象归属
→ 星兽生命原型
```

```text
出生日期与时辰
→ 农历年月日 + 地支时数
→ 卦以八除
→ 母码原始生命倾向
```

二者的世界观关系是：

```text
星兽 = 这个生命本身
母码 = 这个生命来到现实时携带的原始倾向
```

二者的工程关系是：

```text
分别推导
→ 保留各自来源追踪
→ 在视觉表达层合流
```

四象方向和 `symbolicTrigram` 不得替代八除结果。母码也不得反向决定用户的星兽。

“青龙入震”“白虎入兑”等语言只描述生命原型进入母码视觉结构，不描述计算来源。

## 07｜母码与卦象

母码在成卦链中承担下卦位置：

```text
母码 → 下卦
压力种子分类 → 上卦
下卦 + 上卦 → currentHexagramProfile
```

语义边界必须保持：

- 母码是原始生命倾向；
- 上卦表达当前环境力量如何成局；
- `currentHexagramProfile` 是本局变化结构；
- `currentHexagramProfile` 不是最终结果；
- Crystal 是经历和选择之后留下的生命印记。

母码不会因为本局卦象变化而被重新计算。本局变化也不能反向篡改母码来源。

## 08｜原始倾向与现实压力

母码本身不是问题。

压力种子也不是用来证明母码缺陷的证据。

同一种原始生命倾向，在不同环境力量中可能呈现不同姿态：

```text
原始生命倾向
→ 遇到现实引力
→ 在六维生命界面中产生体验
→ 看见倾向如何被使用或过度使用
→ 重新取得选择空间
```

因此，既有字段：

- `baseDrive`；
- `shadowInertia`；
- `pressureEntry`；
- `defaultReactionChain`；
- `unlockPotential`；
- `personalityAsset`；

可以继续作为工程兼容字段，但不得让前台形成以下结论：

- 母码定义了完整人格；
- 用户有一个需要被修好的阴影；
- 某一种母码比另一种更成熟；
- 用户必须升级成系统指定的版本。

更准确的解释是：同一股生命力量，在不同经历中会有不同使用方式。

## 09｜母码前台语言

前台应该说：

- “这是你带来的一种原始倾向。”
- “你最自然地会从这里开始。”
- “压力来时，这股力量可能会收得更紧。”
- “你可以看见它，并选择如何使用它。”

前台不应该说：

- “你就是这种人。”
- “你的性格决定了你会这样。”
- “这是你的缺陷。”
- “这是系统对你的精准分析。”
- “你的母码等级更高。”
- “AI 已经算出真实的你。”
- “你必须完成转化才能成为更好的自己。”

母码语言必须保留人的解释权。系统可以提供镜面，但不能宣布唯一答案。

## 10｜母码卡表达

母码卡不是人格证书，也不是测试报告。

母码卡正面应让用户辨认：

> “原来我常常从这里开始。”

母码卡背面应帮助用户观察：

- 这股生命倾向如何自然展开；
- 它在现实压力下如何收紧；
- 它如何通过身体、情绪、思想、行动、记忆与动机被体验；
- 用户仍然拥有怎样的选择空间。

卡面不得把 `motherCodeTitle` 变成终身身份，不得把 `personalityAsset` 变成价值证明，也不得把 `shadowInertia` 变成缺陷诊断。

现有卡面与文案不在本刀改动范围内。后续如需校准，必须先做前台消费者审查，再单独施工。

## 11｜与 AI 的关系

AI 不生成母码，也不根据对话内容修改母码。

AI 可以帮助用户理解母码在当前经历中的可能表达，但必须遵守：

1. 不把可能性说成事实；
2. 不把倾向说成命定人格；
3. 不替用户解释全部人生；
4. 不利用母码制造依赖；
5. 不让系统输出高于用户自身感受。

母码是观察入口，不是 AI 获得用户解释权的许可证。

母码语义优先级：

## 15｜最终定义

母码不是系统对人的定义。

母码是一个生命来到现实世界时，最先携带并最自然使用的一股力量。

它有来处，但不是命运。

它会影响生命如何开始，但不会决定生命最终成为谁。

观爻通过母码帮助用户看见：

> 我带着怎样的力量来到这里，以及今天我可以怎样更自由地使用它。
