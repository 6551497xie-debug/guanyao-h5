# GUANYAO Genesis Experience Runtime Integration Review Protocol

`RC-GUANYAO-GENESIS-EXPERIENCE-RUNTIME-INTEGRATION-REVIEW-P9`

## 目的

本协议审查 P1–P7 Genesis 视觉状态如何形成一次连续的体验运行链。它只冻结运行顺序、时间轴、转场与用户参与边界，不执行正式 Runtime 接入。

系统约束体验；体验不得反向修改世界观、生命系统、Identity、Engine、Scene Model 或 RenderPlan。

## 七层体验运行链

```text
月｜进入
  ↓
星｜发现
  ↓
时｜交付
  ↓
象｜显影
  ↓
卦｜印记
  ↓
力｜苏醒
  ↓
兽｜归来
```

P7.1「生命存在感」属于兽层归来的存在状态，不新增第八个体验层。

## 用户参与节点

Genesis 只保留一个主动参与点：

```text
交付生命时序
```

月、星、象、卦、力、兽均由体验链自动显化。用户不选择四象、不调整星兽、不选择结果、不进行收藏或宠物化交互。观爻的动作是发现，不是创造。

## 运行节奏

| 阶段 | 用户状态 | 运行语义 |
| --- | --- | --- |
| 月 | 观察 | 进入太阴之境 |
| 星 | 观察 | 星河秩序展开 |
| 时 | 交付 | 将生命时序交给星河 |
| 象 | 观看 | 生命形态由星辰关系显影 |
| 卦 | 观察 | 变化规律形成印记 |
| 力 | 感受 | 生命动力苏醒 |
| 兽 | 认领 | 生命形态安静归来 |

## 转场因果

- **月 → 星**：月华照见星河。
- **星 → 时**：时间进入已有秩序。
- **时 → 象**：时序改变星辰关系。
- **象 → 卦**：变化沉积印记。
- **卦 → 力**：规律唤醒动力。
- **力 → 兽**：动力形成生命存在。

## Runtime 边界

Genesis Runtime 未来只负责：

```text
视觉状态编排
↓
时间轴
↓
转场
↓
用户体验
```

本审查不负责生命计算、身份生成、命理解释、Renderer Command、Storage 状态或正式用户流程。不得提前接入 Reality Pressure、Gravity、Choice、Crystal。

明确：本刀不接入 Reality Pressure、Gravity、Choice、Crystal。
