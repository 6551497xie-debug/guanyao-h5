# GUANYAO Genesis Personal Star Beast Presence Calibration Protocol V1.0

> RC-GUANYAO-GENESIS-PERSONAL-STAR-BEAST-PRESENCE-CALIBRATION-P7.1

## 1. 语义层定位

本刀是 Genesis 第七层「兽｜个人星兽归来」的二阶段生命存在感校准。它承接 Personal Star Beast Reveal Visual State，把已有星辰生命结构推进为“正在存在”的视觉状态；不重新设计星兽，也不改变任何身份或生命计算。

```text
月 → 星 → 时 → 象 → 卦 → 力 → 兽
```

输入是 Reveal Visual State，输出是 Life Presence Visual State。系统约束局部，视觉只显化语义。

## 2. 校准目标

本刀不是增加更多东西，而是让已有结构成为生命：

- Life Orientation：形成主轴、前后关系、生长趋势与姿态倾向，但不出现头部、四肢或动物结构。
- Life Orientation 的视觉验收是主轴形成、前后关系、生长趋势与姿态倾向；不出现头部、四肢或动物结构。
- Stellar Body Cohesion：让星骨连接、粒子聚合、边界形成与局部密度变化共同形成星辰身体，但不模拟肉体、毛发或写实生物。
- Breathing Rhythm：使用极慢周期、核心响应与整体微变化表达生命呼吸；不使用闪烁、脉冲特效或能量爆发。
- Companion Presence：保持静默陪伴、稳定停驻与用户空间关系；不加入表情、对话或宠物行为。

## 3. 六层归来关系

```text
月：提供时间背景
↓
星：提供生命空间
↓
时：提供降临节律
↓
象：提供形态倾向
↓
卦：提供变化纹理
↓
力：提供运动方式
↓
兽：形成存在
```

兽层不能独立设计。Presence Mapping 只增强显化表达，不反向改变上游来源。

## 4. 视觉限制

严格禁止青龙/白虎/朱雀/玄武轮廓、动物脸部、毛发、爪牙、游戏 Boss 感、神兽收藏感与爆发登场动画。目标是东方星辰生命体：神秘、克制、有方向、会呼吸、能陪伴。

## 5. 工程边界

Presence Mapping 只消费 Reveal Visual State，不调用 Engine、Identity Source、MotherCode、LifeArchetype、Scene Model、RenderPlan 或 Renderer。不接 Reality、Gravity、Choice、Crystal、Storage、UI 或 Production。

状态不得包含身份 ID、星宿名、四象名、MotherCode ID、LifeArchetype ID 或动物类型。

## 6. 第一验收

- 是否从星图进入生命？
- 是否感受到主体方向、呼吸与陪伴？
- 是否仍保持神秘与克制？
- 是否没有动物化？

最终判断：它是否从一套星辰结构，成为一个正在存在的生命。
