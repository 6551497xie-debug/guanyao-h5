## 01. What Is Guanyao Persona Behavior Dynamics?

Guanyao is not:

- a personality test;
- a personality label system;
- a result explanation system;
- a prediction system;
- a task system;
- a coaching checklist;
- a pet companion product;
- a 384 yao device.

Guanyao is:

> An AI persona behavior dynamics system that dynamically identifies the user's running persona model under real pressure, helps the user see the old response, complete one small model revision, and deposit that change as a trace of persona evolution.

In Chinese product language:

> 观爻通过现实压力动态识别用户正在运行的人格模型，让用户看见旧运行方式，完成一次微小模型修正，并将变化沉积为人格演化轨迹。

## 02. Core User Value

The user does not come to Guanyao to get:

- a hexagram;
- a card;
- an analysis report;
- a personality label;
- a prediction.

The user comes because real pressure appears and they want to know:

- Why do I react this way?
- Which old model is taking me over?
- Can I respond differently this time?

Guanyao's value is not that it explains the user.

Guanyao's value is that it lets the user see a running model and move one response.

## 04. Core Model Layers

### Layer 1: Six-Dimension / Six-Yao Structure

Six dimensions are the six operating spaces of persona under pressure:

- body;
- emotion;
- thought;
- action;
- memory;
- motivation.

Six yao are the six model-change stages:

- First Yao: Trigger;
- Second Yao: Takeover;
- Third Yao: Explanation;
- Fourth Yao: Solidification;
- Fifth Yao: Seeing;
- Upper Yao: Revision.

Together:

```text
6 spaces x 6 yao stages = 36 potential persona transmission units
```

### Layer 2: 36 Persona Transmission Units

36 units are not frontend content.

They are not 36 tips, 36 prompts, 36 tasks, or 36 reports.

They are the potential backend/product model of how a persona model can move under pressure.

Each unit must include:

- persona space;
- yao stage;
- pressure entry;
- old persona model;
- inertia pattern;
- user-facing seeing language;
- revision direction;
- micro action;
- starbeast change;
- crystal trace.

### Layer 3: PersonaTransmissionUnit

`PersonaTransmissionUnit` is the specific unit hit in the current round.

It is:

> one old persona model running inside one persona space, at one change stage, under one real pressure, with one possible revision direction.

A current session may visually pass through six nodes, but the product meaning comes from the unit that is most relevant to this round.

### Layer 4: Semantic Mapping

Engine language must be translated before it reaches the user.

The translation path is:

```text
Engine fields
↓
Old model explanation
↓
Real-life experience language
↓
Seeing language
↓
Revision direction
↓
Starbeast change
```

Frontend copy must not directly expose:

- `pressureReading`
- `inertiaSignal`
- `antiInstinctHint`
- `cutPotential`
- `interventionPotential`
- `userAgency`
- `mainCut`
- `rootCut`

The frontend should show what the user can recognize, not what the engine calculated.

### Layer 5: Starbeast Manifestation

The starbeast is not a pet.

The starbeast is not a reward mascot.

The starbeast is the lifeform manifestation of the user's persona model in motion.

It should express:

- pressure entering;
- old model tightening;
- seeing becoming possible;
- revision beginning;
- trace being deposited.

In 1.0 this can remain light and restrained.

It does not require complex animation.

## 05. Current-Round Unit Lifecycle

Every active `PersonaTransmissionUnit` follows:

```text
Trigger
↓
Hit
↓
Manifestation
↓
Seeing
↓
Revision
↓
Deposition
```

### Trigger

Reality pressure enters.

### Hit

The system identifies the most relevant persona space and yao stage.

The hit is driven by:

- pressure seed;
- mother code;
- current hexagram orientation;
- `YaoTransmissionProfile`.

### Manifestation

The old model becomes visible through:

- dimension;
- yao stage;
- user-facing insight;
- starbeast state.

### Seeing

The user recognizes:

> This is how I have been protecting myself.

### Revision

The user confirms one small new response.

This is the current-session revision action.

### Deposition

The model movement becomes:

- current-round 64 hexagram-code crystal;
- Personality Ring Lite star point;
- future personality ring trace.

## 06. Product Experience Rule

The user should not experience Guanyao as:

```text
AI gives me an analysis.
```

The user should experience Guanyao as:

```text
I saw how I was moving.
I changed one response.
This round left a trace.
```

Therefore, every production screen should ask:

- What is the user seeing here?
- What old model is becoming visible?
- What small revision is possible?
- What trace will this leave?

If a screen cannot answer these questions, it is probably decorative, generic, or premature.

## 07. 64 Hexagram-Code Meaning

The 64 hexagram code is not:

- prediction;
- personality label;
- final judgment;
- old R8 asset;
- archive item;
- 384 yao device.

The 64 hexagram-code crystal is:

> the current-round staged persona form after the user has seen one old model and confirmed one small revision.

It answers:

```text
What did this round become after I moved one old response?
```

It should not appear as the final asset before:

- pressure has entered;
- current orientation has been established;
- six-dimension / six-yao transmission has completed;
- current-session revision action has been confirmed.

## 08. Personality Ring Meaning

The personality ring is not:

- a full archive;
- a collectible inventory;
- a behavior tracking database;
- a social profile.

In 1.0, Personality Ring Lite is:

> a minimal trace of the user's current-round crystal.

It records that one round left one star point.

Future versions may expand this into:

- multi-round traces;
- beast marks;
- light tracks;
- richer persona evolution maps.

## 11. Construction Rule For Future Work

Every future feature related to `/dynamics` must answer:

1. Which `PersonaTransmissionUnit` is active?
2. What pressure triggered it?
3. What old model did it expose?
4. What can the user recognize?
5. What micro action can the user confirm?
6. What starbeast change expresses this movement?
7. What trace does it leave in the 64 hexagram-code crystal and personality ring?

If it cannot answer these, it should not enter the core 1.0 chain.

> A system that uses real pressure to dynamically hit a persona transmission unit, reveal an old operating model, guide one small response revision, and deposit the movement as a current-round hexagram-code crystal and personality ring trace.

Short version:

> One pressure.
> One old model seen.
> One response moved.
> One trace deposited.
