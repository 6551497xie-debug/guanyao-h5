# GUANYAO Persona Model Revision Protocol

## 01. Core Definition

Guanyao is not a personality test.

Guanyao is not a personality label generator.

Guanyao is not a result explanation system.

Guanyao is a personality behavior dynamics system that helps the user see an old operating model after real pressure enters, and complete one small model revision.

Core sequence:

```text
Reality pressure
-> Old persona model activates
-> Six-dimension / six-yao manifestation
-> Inertia model is exposed
-> Current-session revision action
-> New response forms
-> Current hexagram-code crystal
-> Personality ring deposition
```

The product must follow this order: first the model, then the interaction; first the change logic, then the visual expression.

## 02. Six Dimensions Redefined

The six dimensions are not:

- six feedback modules;
- six page steps;
- six operation buttons.

The six dimensions are six operating spaces of personality under real pressure.

### Body Space

Core question:

How does pressure enter the body?

The body space observes how pressure first appears as contraction, fatigue, heaviness, pain, breath change, or physical alert.

### Emotion Space

Core question:

How does emotion make judgments on behalf of facts?

The emotion space observes how fear, shame, anger, anxiety, grievance, or attachment begins to define the situation before facts are fully seen.

### Thought Space

Core question:

How does the mind explain pressure?

The thought space observes how the mind creates a story, proof, excuse, prediction, or self-persuasion to protect the old model.

### Action Space

Core question:

How does the old model become behavior?

The action space observes where the old reaction becomes delay, avoidance, compensation, over-control, silence, withdrawal, or impulsive action.

### Memory Space

Core question:

How does the past affect the present?

The memory space observes how old experience, old injury, old success, or old fear overlays current evidence.

### Motivation Space

Core question:

What am I really trying to protect?

The motivation space observes the hidden object being protected: safety, dignity, relationship, control, belonging, direction, or survival.

## 03. Six Yao Redefined

The six yao are not progress.

The six yao are not node numbers.

The six yao are stages in the change of a personality model.

### First Yao: Trigger

Pressure enters.

The old model is not yet visible, but the first signal appears.

### Second Yao: Takeover

The old reaction begins to control the field.

The user may feel that the response is automatic.

### Third Yao: Explanation

Thought gives meaning to the pressure.

The mind starts to justify, predict, defend, or delay.

### Fourth Yao: Solidification

The pattern becomes behavior.

The old model is no longer only internal; it starts shaping action.

### Fifth Yao: Seeing

The user begins to see the old model.

This is the first moment where change can become conscious.

### Upper Yao: Revision

A new response becomes possible.

The user does not need to change the whole personality; only one response in this session needs to move.

## 04. Thirty-Six Transmission Units

The Guanyao six-dimension / six-yao model contains:

```text
6 spaces x 6 yao positions = 36 transmission units
```

Every future six-dimension interaction must originate from these units.

Each unit must contain:

1. Space.
2. Yao position.
3. Pressure entry mode.
4. Old persona model.
5. Inertia reaction.
6. User-facing manifestation language.
7. Anti-instinct direction.
8. Current-session revision action.
9. Starbeast change mapping.

Node copy must not be generated from nowhere.

Every node must answer:

- which space is active;
- which yao stage is active;
- what old model is being exposed;
- what anti-instinct direction is possible;
- what small revision action can happen now.

## 05. Engine Field Mapping

The existing causal engine already contains the minimum semantic foundation for this protocol.

`YaoTransmissionProfile` maps to the old-model recognition layer:

- `pressureReading`: how pressure enters this layer.
- `motherCodeInfluence`: how the mother code shapes the old reaction.
- `hexagramInfluence`: how the current hexagram structure affects this layer.
- `transmissionReading`: how the pressure is transmitted through this layer.
- `inertiaSignal`: the old persona model's automatic response.
- `antiInstinctHint`: the anti-instinct direction.
- `cutPotential`: whether this layer can become a cut point.
- `interventionPotential`: how strong the intervention opportunity is.
- `userAgency`: whether the user can realistically act here.

`mainCut`:

The main correction entrance of the current session.

`secondaryCut`:

An auxiliary understanding layer. It can help explain the session, but should not compete with the main correction action in 1.0.

`rootCut`:

The deeper protection mechanism. It explains what the old model is trying to protect, but it should not become a heavy behavior system in 1.0.

## 06. Current-Session Revision Action Protocol

The current-session revision action is not:

- a task;
- a choice game;
- a behavior check-in;
- a growth score;
- a personality test result.

It is:

The user's active confirmation of one new response direction after seeing the old model.

Required position:

```text
Six-dimension / six-yao manifestation complete
-> Current-session revision action
-> Current-session crystal
```

The action must be:

- small;
- concrete;
- executable;
- connected to the current cut point;
- free from shame, punishment, or self-optimization pressure.

The action must not ask the user to change their whole personality.

It only asks:

What is one old reaction that can move in this session?

## 07. Starbeast Mapping Protocol

The starbeast is not a companion character.

The starbeast is not a pet.

The starbeast is not an external role.

The starbeast is the life manifestation of the user's persona model after change.

Dimension mapping:

| Dimension | Persona Model Change | Starbeast Mapping |
| --- | --- | --- |
| Body | From contraction to carrying | Beast body stabilizes or releases contraction |
| Emotion | From emotional takeover to named feeling | Light field becomes clearer |
| Thought | From scattered explanation to focused judgment | Light tracks gather or re-route |
| Action | From blocked or automatic behavior to a new direction | Direction line shifts or opens |
| Memory | From old trace covering the present to old trace being seen | Old marks surface without covering the body |
| Motivation | From hidden protection to conscious protection | Star core becomes visible |

This protocol does not require complex animation in 1.0.

It only defines how model change should later affect visual language.

## 08. Hexagram Code Redefined

The 64 hexagram code is not prediction.

The 64 hexagram code is not a label.

The 64 hexagram code is not a result report.

The 64 hexagram code is a staged persona form after the user completes one persona model revision in the current session.

Relationship:

- Mother code: original model.
- Current hexagram orientation: current position.
- Six-dimension / six-yao: model expansion.
- Revision action: model update.
- Hexagram code: updated session form.

Therefore the hexagram-code card must not appear as a final asset before the six-dimension / six-yao process and current-session revision action are complete.
