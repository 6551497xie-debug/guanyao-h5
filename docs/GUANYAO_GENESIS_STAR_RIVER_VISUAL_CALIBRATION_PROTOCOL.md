# RC-GUANYAO-GENESIS-STAR-RIVER-VISUAL-CALIBRATION-P2

# GUANYAO Genesis Star River Visual Calibration V1.0

本刀把 Genesis 第二层「星｜星河秩序」落为视觉状态与映射契约，只消费 P1 Moon Origin Visual State，不改变任何生命计算或正式产品流程。

## 层级范围

```text
月｜太阴入口
↓
星｜星河秩序
```

星层只处理月华照见后的星河秩序，不提前进入时、象、卦、力、兽。

## Star River Visual State

- `starFieldPresence = STELLAR_FIELD_PRESENT`
- `stellarOrderExpression = ORDERED_STELLAR_RELATIONS`
- `mansionGroupExpression = SEVEN_POINT_GROUP_TENDENCY`
- `celestialMotionRhythm = SLOW_CELESTIAL_DRIFT`
- `spatialDepthExpression = DEEP_LAYERED_COSMOS`
- `observationState = QUIET_OBSERVATION`

星河存在表达月华照亮后的巨大宇宙；星辰以关系、微弱轨迹、群体运动和星河流转表达秩序，让用户感到星河有序，而不是随机粒子、科幻 HUD、线框图或知识展示。

七宿结构在本层只作为星群聚合趋势、七点关系和结构暗示，禁止直接形成四象、动物或显示宿名。

## 月→星关系

```text
月华
↓
照见
↓
星河秩序
```

月是时间入口，星是秩序显现。星层不得替代月层，也不得成为独立宇宙入口。

## 速度与节奏

继承 P1 的慢节律：星尘缓慢漂移，星河缓慢运行，空间变化不可突兀。禁止快速飞行；禁止粒子爆发；禁止穿越特效和科幻跃迁。目标是让用户感到“宇宙正在运行”，而不是“动画正在播放”。

## 严格边界

Star River Visual State 不包含 Identity、Birth Time、Mansion Result、Four Symbol Result、MotherCode 或 Personal Star Beast。

映射服务只消费 Moon Origin Visual State，不调用 Engine、Identity Source、Scene Model、RenderPlan 或 Renderer，不污染时、象、卦、力、兽层。

本刀不修改月层语义、生命计算、星宿算法、四象、卦象、MotherCode、Personal Star Beast 或真实用户流程。
