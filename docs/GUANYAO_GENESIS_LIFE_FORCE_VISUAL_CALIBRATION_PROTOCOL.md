# GUANYAO Genesis Life Force Visual Calibration Protocol V1.0

> RC-GUANYAO-GENESIS-LIFE-FORCE-VISUAL-CALIBRATION-P6

## 1. 语义层定位

本刀属于 Genesis 七层视觉语义中的「力｜生命原力」。它只表达原力如何表现，不展示 MotherCode、LifeArchetype 分类，也不改变任何生命来源。

前置关系固定为：

```text
月 → 星 → 时 → 象 → 卦 → 力 → 兽
```

P6 的输入是 P5 Hexagram Visual State，输出是 Life Force Visual State。视觉只显化语义，不创造生命。

## 2. 生命原力定义

力不是添加。力是生命原本携带的动力方式开始苏醒。

它让用户感受到：**生命开始以自己的方式运行**。原力通过节律、运动倾向、星核唤醒和整体结构响应表达，而不是通过属性、数值或技能效果表达。

## 3. 四种表达方向

### 节律变化

允许呼吸频率变化、聚合节奏变化、光流节奏变化、结构响应变化；不使用技能释放、能量爆发或光柱效果。

### 运动方式变化

允许流动方向变化、展开/收束变化、稳定/突破变化、内聚/外放变化；不表现攻击动作、战斗效果或游戏技能。

### 核心状态变化

允许星核呼吸变化、内部光流变化、结构稳定变化。核心是既有生命中心的苏醒，不生成新核心，也不表达属性强化。

### 生命结构影响

允许星骨张力变化、形态场变化、空间关系变化。原力影响表达，不改变身份、象或卦。

## 4. 卦与力的边界

```text
卦 → 变化规律 → 力 → 动力方式
```

卦回答：如何变化。力回答：以什么方式变化。

P6 不反向解释卦，也不提前进入个人星兽。完整汇合关系为：

```text
象 + 卦 + 力
        ↓
个人星兽
```

本刀在「力」处停止。

因此本刀不提前生成星兽，不改变卦层；不修改月层、星层、时层或象层，也不修改卦层。

明确边界：不修改月层；不修改星层；不修改时层；不修改象层；不修改卦层。

## 5. 工程边界

Life Force Visual State 只消费 Hexagram Visual State。它不调用 MotherCode Engine、LifeArchetype Engine、Identity Source、Scene Model、RenderPlan 或 Renderer；不包含 MotherCode ID、LifeArchetype ID、八母码名称、Identity 或 Personal Star Beast。

本刀不修改月层、星层、时层、象层、卦层，不修改 MotherCode 计算、LifeArchetype 计算或正式 Personal Star Beast。

## 6. 验收

- 用户是否感觉生命开始拥有动力，而不是技能释放？
- 是否没有人格标签感，且保持东方生命观？
- 是否自然期待下一阶段显化？
- 是否确认未提前生成星兽、未改变卦层？
