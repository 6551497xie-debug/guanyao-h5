## 一、核心定性：观爻 2.0 不是网页

观爻 2.0 不是网页。

它是一套 **1px 一字线驱动的行为物理系统**。用户感知到的不是页面，而是一个有质量、有阻尼、有边界感的空间。所有 UI 决策均服从这一定性，违反此定性的方案一律拒绝。

---

## 二、全流程一屏锁定：No Scrolling

- 任何页面、任何状态下，**禁止出现纵向或横向滚动条**。
- 所有内容必须在单屏视口内完整呈现。
- 过渡动效（滑入、翻转、消融）不等于滚动；动效服务于物理感，而非内容溢出的补偿。
- 若内容无法在单屏内容纳，优先裁剪内容，而非引入滚动。

---

## 三、消灭网页控件

以下元素在观爻 2.0 中**永久禁止出现**：

| 禁止项 | 替代思路 |
|--------|----------|
| 闭合方框（border box） | 1px 一字线划定区域边界 |
| 圆角胶囊按钮 | 线性触发区 / 长按区域 |
| 单选框（radio button） | 滑动选择 / 物理拨片 |
| 下拉框（select / dropdown） | 上滑抽屉 / 横向滑轨 |
| 网页 tags / chips | 爻线标注 / 字符行内嵌入 |

一旦设计稿出现上述元素，视为未达标，须重做。

## 四、1px 一字线：生命监测仪与世界边界

1px 一字线是观爻 2.0 的核心视觉语言，承担双重职责：

**① 生命监测仪**  
线的状态（静止 / 震动 / 断裂 / 复归）反映系统当前的能量状态与用户行为反馈。线不是装饰，是系统的呼吸。

**② 世界边界**  
线划定空间的边缘与层级。线内是当前焦点，线外是未激活态。用户通过与线的交互（触碰、穿越、拉伸）完成操作，而非点击传统控件。

线的物理属性：
- 宽度：严格 1px，不可模糊为 2px 或渐变宽度
- 颜色：遵循系统 token，禁止随意使用纯黑或纯白
- 行为：可震动、可断裂、可复归，由 Canvas 表现层驱动

---

## 五、Canvas 只接表现层，不动底层因果链

**Canvas 的职责边界：**

- Canvas 负责：线的渲染、动效、物理模拟、粒子、阴影等所有视觉表现
- Canvas 不负责：卦象计算、压力种子逻辑、六维空间投影、LocalStorage 读写、路由跳转

**旧 DOM 逻辑保留原则：**

- 现有 DOM 事件逻辑（点击、表单、路由）**一律保留，不得删除**
- Canvas 层通过 `trigger` 机制调用旧有 DOM 事件，形成「Canvas 触发 → DOM 响应」的单向驱动链
- Canvas 层是神经，DOM 层是肌肉，Codex 是骨骼——三者分层，不得越权

---

## 六、四段视觉链

观爻 2.0 的全流程分为四段视觉链，每段有独立的物理感知目标：

### 第一段：见自己
- 用户进入系统的第一接触面
- 视觉目标：安静、收敛、有重量感
- 交互目标：让用户感知到「系统在看我」，而非「我在填表」
- 禁止出现任何引导性文字气泡、onboarding tooltip

### 第二段：观变化
- 卦象生成与展示阶段
- 视觉目标：动态、有因果感、线在生长
- 交互目标：用户感知到卦象是「算出来的」而非「选出来的」
- 爻线生成须有时序动效，禁止瞬间全显

### 第三段：循规律
- 卦象解读与六维空间呈现阶段
- 视觉目标：层次分明、信息有密度但不压迫
- 交互目标：用户能自主探索各维度，而非被动阅读
- 六维空间投影图必须服从 token 系统，不得随意改色

### 第四段：破心结
- 资产沉积与压力种子激活阶段
- 视觉目标：仪式感、终结感、有沉淀重量
- 交互目标：用户感知到「这一次已经留下痕迹」
- 禁止出现「完成」「提交」等网页化措辞

---

## 七、声音与马达阻尼：增强层规范

声音和马达振动作为**可选增强层**，遵循以下规则：

- **有权限时**：线的震动、断裂、复归配合触觉反馈（Haptic）和音效
- **无权限时**：静默降级，视觉表现完整保留，不出现任何权限请求弹窗
- **降级不得影响功能**：所有核心交互在无声音、无振动的情况下必须完整可用
- 音效设计原则：低频、短促、非音乐性，服务于物理感而非情绪渲染

---

## 八、母码卡与卦码卡：分开重构

母码卡和卦码卡是两套独立的视觉系统，禁止共用模板。

### 母码卡（Mother Code Card）
- A 面：**严禁出现卦符**，只呈现母码编号、线性元素、能量状态指示
- B 面：允许固定六爻卦形，作为深度信息层
- 母码卡的视觉语言：稳定、基础、无变化感

### 卦码卡（Hexagram Card）
- A 面：**严禁出现卦符**，呈现卦名、核心字词、线性动效
- B 面：允许固定六爻卦形，附爻辞摘要
- 卦码卡的视觉语言：动态、有变化感、响应用户当前状态

**A 面不得出现卦符**是硬性约束，不因设计需要而妥协。
