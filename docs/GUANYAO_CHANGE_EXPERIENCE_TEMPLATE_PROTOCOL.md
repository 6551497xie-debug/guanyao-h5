# GUANYAO Change Experience Template Protocol

## 00. Document Position

This document abstracts the first complete Guanyao 1.0 self-change golden sample from:

```text
action-five-awareness
```

It defines the reusable experience template behind:

```text
see
-> understand
-> choose
-> move
-> leave a trace
```

It does not define a new engine, a new 36-unit library, a new storage schema, or a new route.

It should be read together with:

- `GUANYAO_PERSONA_BEHAVIOR_DYNAMICS_CORE_V1.md`
- `GUANYAO_PERSONA_TRANSMISSION_UNIT_PROTOCOL.md`
- `GUANYAO_PERSONA_TRANSMISSION_EXPERIENCE_CONSUMPTION_CONTRACT.md`
- `GUANYAO_HEXAGRAM_CRYSTAL_MIGRATION_CAUSAL_PROTOCOL.md`

## 01. Core Definition

The Guanyao change experience template is:

> A current real pressure enters the user's mother-code base, becomes a current-round situation, passes through the user's six persona spaces, reveals one old protective response, lets the user confirm one new response, and deposits the movement as a current-round crystal imprint.

In Chinese product language:

> 一个现实压力进入母码底盘后，形成本局处境；用户在六维中看见旧反应如何运行，理解它曾经如何保护自己，确认一次新的回应，并让这次移动沉积为本局变化印记。

The template is not:

- a personality test;
- a diagnosis;
- an analysis report;
- a task flow;
- a card-generation shortcut;
- a reward loop.

## 02. Fixed Experience Spine

Every complete Guanyao 1.0 self-change experience should follow this spine:

```text
1. Pressure enters
2. Current situation forms
3. Six dimensions manifest
4. Old response is recognized
5. Protection meaning is understood
6. User chooses a new response
7. Transformation Moment
8. Growth meaning
9. Change imprint
10. Hexagram-code crystal
```

Chinese frontend semantics:

```text
压力进入
↓
本局定位
↓
六维显影
↓
旧反应识别
↓
保护机制理解
↓
用户选择新回应
↓
改变发生
↓
成长价值
↓
变化留痕
↓
卦码结晶
```

No production chain should skip directly from pressure to crystal.

No production chain should issue a 64 hexagram-code crystal before the user has seen the old response and confirmed a new response.

## 03. Template Structure

The reusable template is:

```text
Pressure Context
↓
Current Situation
↓
Old Reaction
↓
Protection Meaning
↓
New Response
↓
Transformation Moment
↓
Growth Meaning
↓
Crystal Imprint
```

### 1. Pressure Context

Answers:

```text
What real pressure is entering this round?
```

It must feel current and concrete.

It should not sound like:

- a generic example;
- a system case;
- a psychological category;
- a quiz result.

Golden sample reference:

```text
Facing a situation that must move forward while the outcome remains uncertain.
```

Chinese experience direction:

```text
面对必须推进、但结果仍不确定的现实局。
```

### 2. Current Situation

Answers:

```text
Where am I now?
```

It is expressed through current-round orientation.

It should feel like:

```text
I have entered a situation.
```

It should not feel like:

```text
The system calculated a fortune.
```

Golden sample reference:

```text
You are pushed into a position where direction must be chosen again.
```

### 3. Old Reaction

Answers:

```text
What old response starts running under this pressure?
```

It must describe a protective operating pattern, not a personality label.

Bad:

```text
You are action-oriented.
```

Good:

```text
When the result is uncertain, you first use action to pull the situation back into control.
```

Chinese golden sample:

```text
结果还不确定时，你会先用行动把局面拉回掌控。
```

### 4. Protection Meaning

Answers:

```text
Why did this old response once protect me?
```

This is the difference between Guanyao and a report.

The user should feel:

```text
I was not wrong. I was using an old protection method.
```

The copy should avoid blame.

Golden sample direction:

```text
Action is an ability. This time, it needs judgment before movement.
```

Chinese golden sample:

```text
行动力是能力，现在需要先判断再出手。
```

### 5. New Response

Answers:

```text
What small response can I choose differently now?
```

It must be:

- small;
- current-round;
- specific;
- confirmable by the user.

It must not be:

- a task;
- a habit plan;
- a score;
- a long-term goal;
- an answer forced by the system.

Golden sample:

```text
Before rushing to push forward next time, first confirm the real problem to solve.
```

Chinese golden sample:

```text
下一次急着推进前，先确认真正要解决的问题。
```

### 6. Transformation Moment

Answers:

```text
What just moved?
```

This is the moment between confirmation and crystal.

It should clearly show:

```text
Old response
↓
New response
```

The user should feel:

```text
I moved a little.
```

The starbeast can express this movement lightly.

Golden sample:

```text
The starbeast shifts from rushing forward to first stabilizing direction.
```

Chinese golden sample:

```text
星兽从向前急冲，转为先稳住方向。
```

### 7. Growth Meaning

Answers:

```text
What did this movement mean for my growth?
```

Growth Meaning is the user-facing value expression after a persona-dynamics deflection.

It is not:

- a score;
- a level;
- an achievement;
- a game reward;
- a long-term growth metric;
- a storage field.

It is:

> The language by which the user can recognize and own the value of this small movement.

Three language layers must remain separated:

```text
1. Experience language
   User is experiencing:
   "First see the emotion, then confirm the fact."

2. Persona-dynamics migration language
   The system understands:
   "Emotion moves from automatic judgment to observable signal."

3. Growth-value language
   User receives:
   "Emotional awareness upgraded."
```

Growth Meaning is the third layer.

It should not expose engine fields or pretend to be a quantified achievement.

It should sound like a concise value the user can carry away.

#### Growth Meaning Examples

| Space | Old Reaction | New Response | Growth Meaning |
| --- | --- | --- | --- |
| Action | Restore control through action. | Judge first, then act. | Action pattern upgraded. |
| Emotion | Let emotion make the factual conclusion. | See the emotion, then confirm the fact. | Emotional awareness upgraded. |
| Thought | Treat interpretation as reality. | See the fact first, then see the interpretation. | Cognition upgraded. |
| Experience | Let past experience limit the present. | Let experience serve the present. | Experience wisdom upgraded. |
| Motivation | Prove the self through external results. | Return to what truly needs protection. | Inner drive awakened. |

Chinese product shorthand:

```text
行动模式升级
情绪觉察升级
认知升级
经验智慧升级
内在驱动力觉醒
```

### 8. Crystal Imprint

Answers:

```text
What did this movement leave behind?
```

The crystal is not a result issued by the system.

The crystal is:

> The shape left by this round of movement.

Golden sample:

```text
This round, you moved a little from the old response. This new response left its shape.
```

Chinese golden sample:

```text
这一局，你从旧反应中移动了一点。
这次新的回应，留下了它的形状。
```

## 04. Variable Sources

The template is fixed.

The content inside the template is variable.

### A. From Mother Code

Mother code provides:

- original persona base;
- base operating tendency;
- pressure-sensitive zone;
- default protection pattern;
- starbeast / four-symbol source;
- lower-trigram base for current orientation.

It answers:

```text
What base structure is pressure entering?
```

In the template, mother code affects:

- how pressure is framed;
- what old response is likely to appear;
- how the starbeast enters the round;
- how the current situation is oriented.

### B. From Pressure Seed

Pressure seed provides:

- current real situation;
- triggering context;
- uncertainty / conflict / relation field;
- user-facing pressure language;
- what is being activated now.

It answers:

```text
What is pressing the user in this round?
```

In the template, pressure seed affects:

- `Pressure Context`;
- `Current Situation`;
- the concrete tone of `Old Reaction`;
- the user's feeling of "this is my current situation."

### C. From Current Hexagram Orientation

Current hexagram orientation provides:

- current-round position;
- situation image;
- lower / upper trigram relation;
- starbeast entering the current round.

It answers:

```text
Where am I now?
```

It must not be presented as the final crystal.

In the template, it supports:

- `Current Situation`;
- the bridge from pressure to six-dimension experience;
- the feeling that this pressure has formed a round.

### D. From PersonaTransmission

PersonaTransmission provides:

- old model;
- inertia pattern;
- user-facing insight;
- revision direction;
- micro action;
- crystal trace.

It answers:

```text
What old response is running, and how can it move?
```

In the template, it controls:

- `Old Reaction`;
- `Protection Meaning`;
- `New Response`;
- `Transformation Moment`;
- `Growth Meaning`;
- `Crystal Imprint`.

Raw engine fields must not be shown directly.

### E. From Starbeast Feedback

Starbeast feedback provides:

- state before movement;
- state after movement;
- visual / lifeform cue;
- a felt sense of persona movement.

It answers:

```text
How does my inner movement become visible?
```

In the template, it supports:

- six-dimension manifestation;
- old response tightening;
- new response beginning;
- Transformation Moment;
- Growth Meaning when the movement becomes recognizable as value;
- trace deposition.

The starbeast must remain:

```text
persona movement made visible
```

not:

```text
pet
companion
reward mascot
```

## 05. Six-Dimension Role Inside the Template

The six dimensions are not six reports.

They are the user's path of recognition:

```text
body      -> pressure enters sensation
emotion   -> pressure becomes feeling
thought   -> pressure becomes interpretation
action    -> pressure becomes response
memory    -> pressure activates old experience
motivation-> pressure reveals what is being protected
```

The six dimensions should prepare the user for the new response.

They should not exhaust the user with explanations.

Each dimension should keep the rhythm:

```text
manifestation line
↓
understanding line
```

## 07. Boundary

Growth Meaning does not change:

- `PersonaTransmissionUnit`;
- `PersonaMigrationImpact`;
- `CrystalState`;
- the Hexagram Engine;
- 64 hexagram-code generation;
- storage;
- routes.

It is only a user-value expression layer.

It may guide future copy, fixture, and presentation work.

It must not become a score, level, achievement, or reward system.

This template does not require:

- expanding the 36-unit model;
- adding a pressure library;
- adding a storage field;
- changing `PersonaTransmissionUnit`;
- changing `PersonaMigrationImpact`;
- changing `CrystalState`;
- changing the Hexagram Engine;
- changing 64 hexagram-code generation;
- changing routes.

It is a product experience protocol.

It should guide future copy, fixture, visual, and UI work only after the core Persona Behavior Dynamics model remains intact.
