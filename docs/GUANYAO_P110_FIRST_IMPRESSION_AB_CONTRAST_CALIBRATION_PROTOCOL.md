# GUANYAO P110 First Impression A/B Contrast Calibration Protocol

协议编号：`RC-GUANYAO-GENESIS-FIRST-IMPRESSION-AB-CONTRAST-CALIBRATION-P110`

阶段：`GUANYAO Genesis WebGL Prototype Calibration Phase`

状态：`ISOLATED_PROTOTYPE_ONLY / HUMAN_REVIEW_REQUIRED`

## 01｜本刀目标

P110 校准两个正式生命样本的第一眼差异。

唯一允许的链路：

```text
正式生命身份 Case A
  → Case A Genesis Projection Bundle
  → Case A RenderPlan
  → 同一隔离 Renderer

正式生命身份 Case B
  → Case B Genesis Projection Bundle
  → Case B RenderPlan
  → 同一隔离 Renderer
```

同一套 Manifestation Grammar 与同一个 Renderer，必须消费各自正式来源产生的投影。

## 02｜本刀修正的问题

此前 Harness 的 Case A / Case B 虽然使用不同 RenderPlan，但 Genesis 的本命星宿、四象形态场、母码原力、个人显化和现实压力投影共用 Case A 引用。

这会让“另一种生命”只在部分 RenderPlan 参数上不同，不能作为真实 A/B 第一眼验收样本。

P110 将这些投影按正式 Fixture 分别建立 Bundle：

- Case A：使用 Case A 的 Mansion、Four Symbol、MotherCode、LifeArchetype 与 PersonalStarBeast 引用；
- Case B：使用 Case B 的 Mansion、Four Symbol、MotherCode、LifeArchetype 与 PersonalStarBeast 引用；
- Time Sequence 仍可共享，因为它只表达共同的生命场时间响应。

## 03｜视觉验收目标

A/B 不得只表现为颜色变化，至少应在以下关系中出现结构差异：

1. 生命核心与骨架的组织关系；
2. 骨架方向、分支与支撑关系；
3. 四象形态场对收束、展开、包裹或流动的影响；
4. 现实压力对边界、核心和流向的不同响应。

## 04｜边界

P110 只修正隔离 Harness 的案例投影绑定。

禁止：

- 修改 Identity、Engine、Manifestation Grammar、Scene Model 或 RenderPlan Contract；
- 修改 Renderer 内部组织逻辑；
- 通过视觉标签、动物名称或用户资料制造差异；
- 接入 Production、Launch、Gravity、Dynamics、Crystal 或 Storage。

工程 Gate 只能确认来源绑定完整，最终是否“像另一种生命”仍需人工第一眼验收。
