# GUANYAO 1.0 Official Asset Generation Protocol

This document locks the boundary for the 1.0 official asset generation candidate stage.

It defines when `finalAssetCandidate(FINAL_ASSET_CANDIDATE)` may enter `officialAssetGenerationCandidate(OFFICIAL_ASSET_GENERATION_CANDIDATE)`.

It does not generate a formal 64-hexagram asset card, does not connect to UI, and does not add any route.

## 1. Current Asset Chain

The current P0 asset chain has reached:

```text

→ currentPrimarySpaceId
→ 6节点完成态
→ starbeastFeedbackComplete
```

The next stage may only create an official asset generation candidate:

```text
finalAssetCandidate(FINAL_ASSET_CANDIDATE)
→ resolveOfficialAssetGenerationCandidate()
→ officialAssetGenerationCandidate(OFFICIAL_ASSET_GENERATION_CANDIDATE)
```

## 2. OFFICIAL_ASSET_GENERATION Position

`officialAssetGenerationCandidate` is still not a formal UI card.

It is the final system gate that can allow a later formal asset object generator to run.

It only answers:

- whether this round satisfies formal asset generation conditions;
- which elements must be present at the same time;
- whether legacy routes, legacy asset pages, and commercialization remain forbidden;
- which minimum fields a future formal asset object would need.

It must not directly output:

- formal UI;
- formal React component fields;
- payment fields;
- collection fields;
- unlock fields;
- `route`;
- `archiveRoute`;
- `legacyRoute`.

Formal `hexagramCode`, `hexagramName`, and `cardName` are not generated by this protocol step.

The next resolver may still only output `officialAssetGenerationCandidate`. It must not output the formal card object.

## 5. Readiness Rules

Future resolver rules must obey:

1. If `finalAssetCandidate.readiness !== "READY_FOR_OFFICIAL_ASSET_GENERATION"`, `readiness = "NOT_READY"`.
2. Only when `finalAssetCandidate.readiness === "READY_FOR_OFFICIAL_ASSET_GENERATION"` and all critical `assetElementsReady` conditions are true, `readiness = "READY_FOR_OFFICIAL_ASSET_OBJECT"`.
3. Critical conditions must include at least:
   - `personaSource = true`;
   - `pressureTrace = true`;
   - `primaryDimension = true`;
   - `sixNodeCompletion = true`;
   - `starbeastFeedback = true`;
   - `forceTranslation = true`;
   - `renderBlueprint = true`;
   - `forbiddenLegacyRoute = true`.
4. `noLegacyRoute` must always be `true`.
5. `noCommercialPayload` must always be `true`.
6. `officialAssetObjectRequired` must always be `true`.
7. `forbiddenLegacyRoute` must always be `true`.
8. `commercialPayloadForbidden` must always be `true`.

## 6. Expression Principles

`generationReason` must continue to obey:

- no fortune or misfortune;
- no punishment;
- no failure shame;
- no task feeling;
- no fatalistic pressure;
- no mystical fear;
- no implication that the user has already received a formal asset card;
- no implication of paid unlock;
- no implication that collection is complete.

Allowed expressions:

- "这局已具备进入正式资产对象生成的候选条件。"
- "仍需下一层官方资产对象生成器完成压印。"
- "旧路由与商业载荷仍被禁止。"
- "当前只是正式生成前的系统闸门。"

Forbidden expressions:

- "你获得了一张卦码卡";
- "已解锁";
- "立即收藏";
- "购买";
- 大吉 / 大凶;
- 任务完成;
- 成功打卡;
- 通过考验;
- 能量不足;
- 未完成;
- 失败.
