# GUANYAO 1.0 Protocol Freeze

This document freezes the GUANYAO 1.0 protocol chain as a closed candidate-loop system.

It stops further protocol-index expansion and moves the product from protocol construction into execution convergence.

## 1. Freeze Decision

GUANYAO 1.0 has completed the candidate-state protocol loop.

The current protocol chain is frozen as:

```text
PENDING
→ DRAFT
→ MAPPING
→ FORCE_TRANSLATION
→ ASSET_SHELL
→ CARD_BLUEPRINT
→ ASSET_RENDER_CANDIDATE
→ FINAL_ASSET_CANDIDATE
→ OFFICIAL_ASSET_GENERATION_CANDIDATE
→ OFFICIAL_ASSET_OBJECT
→ ASSET_CARD_RENDER_CANDIDATE
→ ASSET_CARD_UI_CANDIDATE
→ UI_COMPONENT_CANDIDATE
→ COMPONENT_IMPLEMENTATION_CANDIDATE
→ SAFE_COMPONENT_STUB_CANDIDATE
→ SAFE_COMPONENT_STUB_IMPLEMENTATION_CANDIDATE
```

This is now the final protocol-layer endpoint for GUANYAO 1.0.

## 2. Candidate Loop Complete

The protocol layer has already defined:

- pressure seed entry into the causal chain;
- primary petal derivation;
- six-node completion state;
- starbeast feedback;
- asset candidate progression from PENDING through official asset data object;
- asset render candidate boundaries;
- UI candidate boundaries;
- component candidate boundaries;
- safe component stub candidate boundaries;
- safe component stub implementation candidate boundaries.

No additional candidate protocol layer is required before execution convergence begins.

## 3. Two-Layer System

From this point onward, GUANYAO 1.0 is divided into two layers.

### A. Protocol Layer

Status: completed and frozen.

Scope:

- defines the full candidate chain;
- defines forbidden boundaries;
- prevents legacy route regression;
- prevents commercialization from entering early;
- prevents formal hexagram identity fields from appearing before explicit identity assignment;
- prevents real UI or route creation inside candidate layers.

The protocol layer must not continue expanding through more `xxx → resolve → xxxCandidate → P1 index` loops.

### B. Execution Layer

Status: next active construction phase.

Scope:

- starts from `SAFE_COMPONENT_STUB_IMPLEMENTATION_CANDIDATE`;
- converts the frozen protocol chain into a real minimum runnable render convergence path;
- keeps the active `/dynamics` causal chain intact;
- keeps legacy `/hexagram-stamp` isolated;
- avoids adding route, commercialization, collection, payment, or unlock paths unless explicitly unfrozen later.

Execution work must use the frozen protocol layer as constraints, not as a reason to create more protocol layers.

## 4. Only Allowed Next Direction

The only allowed next phase is:

```text
SAFE_COMPONENT_STUB_IMPLEMENTATION_CANDIDATE
→ real minimum runnable render convergence
```

This means future work should begin converting the already frozen candidate structure into the smallest safe runtime/render path.

Allowed execution-phase work may include:

- using existing candidate objects as the data source;
- defining the smallest safe component stub implementation plan;
- preparing a non-route, non-commercial render convergence path;
- verifying that no legacy route or old asset page is reconnected;
- preserving the no-punishment language system.

## 5. Forbidden Expansion

From this freeze onward, GUANYAO 1.0 forbids:

- adding another protocol index layer;
- adding a new `xxx-P1` index bridge;
- adding a new `xxx-P0` protocol expansion only to extend the chain again;
- continuing the infinite pattern of `xxx → resolve → xxxCandidate → P1 index`;
- creating another candidate stage after `SAFE_COMPONENT_STUB_IMPLEMENTATION_CANDIDATE`;
- using "next protocol" as a substitute for execution convergence;
- reopening legacy `/hexagram-stamp`;
- adding formal route, UI page, collection, payment, unlock, or commercial entry during protocol freeze;
- generating formal `hexagramCode`, `hexagramName`, `cardName`, `finalAssetId`, or `officialAssetId` without a separate explicit unfreeze.

## 6. Construction Rule After Freeze

Every future GUANYAO 1.0 request must first be classified as one of:

1. Execution convergence.
2. Runtime verification.
3. UI/visual refinement within the existing active flow.
4. Language refinement within the existing language system.
5. Documentation clarification that does not add a new protocol layer.

If a request attempts to create another protocol stage, another protocol index bridge, or another candidate chain link, it must be rejected or redirected to execution convergence.

## 7. Acceptance

This freeze is active when:

1. `docs/GUANYAO_1_0_PROTOCOL_FREEZE.md` exists.
2. `docs/GUANYAO_1_0_PROTOCOL_INDEX.md` points to this freeze.
3. The protocol index no longer encourages new protocol-layer expansion.
4. The next allowed direction is execution convergence from `SAFE_COMPONENT_STUB_IMPLEMENTATION_CANDIDATE`.
5. No `src` runtime file is changed by this freeze step.
6. No route is added.
7. No real UI is connected.
8. No commercialization is connected.
