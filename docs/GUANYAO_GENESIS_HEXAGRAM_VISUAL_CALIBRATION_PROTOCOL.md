# GUANYAO Genesis Hexagram Visual Calibration Protocol

RC-GUANYAO-GENESIS-HEXAGRAM-VISUAL-CALIBRATION-P5
Genesis 第五层：卦｜变化印记

## 1. 目的与边界

本协议把 Genesis 第五层「卦｜变化印记」落为视觉状态与映射契约，只消费 P4 Symbol Visual State，不改变卦计算逻辑、Hexagram Engine、Identity Source、四象计算、MotherCode 或产品流程。

这里的卦不是一个知识符号，也不是展示一个卦。卦是生命形态运动过程中留下的变化规律结构印记。用户应感受到：变化有规律。

固定上游与下游：

```text
象
↓
运动
↓
观察变化
↓
卦形成
↓
力（后续层，当前不进入）
```

## 2. Hexagram Visual State

`GenesisHexagramVisualState` 只表达：

- 变化规律正在出现；
- 阴阳节律的收缩与展开；
- 六层变化结构的递进；
- 状态之间的转换轨迹；
- 变化沉积为生命印记。

它不包含 Identity、Four Symbol Result、Hexagram Calculation Result、MotherCode、LifeArchetype 或 Personal Star Beast。

## 3. 阴阳节律

阴阳在本层是变化关系，不是黑白二元图、阴阳 Logo 或太极图。允许用收缩与展开、明暗变化、连续与断裂、聚合与分离表达节律。

目标是让用户感受到：变化有规律，而不是进入算命或预测语境。

## 4. 六层结构

六层表达生命变化的层级，可使用六层光纹、六段结构变化或六个变化节点呈现递进关系。禁止直接显示六根爻线、卦卡或命理报告。

“六层”是视觉递进结构，不是 Hexagram Calculation Result 的展示。

## 5. 变化轨迹

轨迹表达从一种状态走向另一种状态，允许结构变化痕迹、转换路径与连续观察感。不预测未来，禁止命运动画或结果预告。

## 6. 印记形成

变化规律被记录为生命印记，允许星纹形成、结构沉积与光痕留下。禁止徽章、勋章与游戏成就化表达。

## 7. 卦与后续原力

卦回答“如何变化”，力回答“以什么力量变化”。本层不提前表达 MotherCode，不生成星兽，不解释生命结果。

本刀不修改月层；不修改星层；不修改时层；不修改象层。

## 8. 工程边界

Hexagram Mapping Service 只负责：

```text
Symbol Visual State
↓
Hexagram Visual State
```

它不调用 Engine、Hexagram Calculation Service、Identity Source、Scene Model、RenderPlan 或 Renderer，不显示计算结果，不写 UI、Storage，也不生产资产。

## 9. 视觉验收

人工验收必须回答：

1. 用户是否感受到变化规律存在？
2. 是否没有进入算命或预测感？
3. 是否没有看到一个知识符号？
4. 是否保持 Genesis 沉浸感？
5. 是否自然承接后续生命原力？

只有卦层验收通过后，才进入 Genesis Life Force Visual Calibration。
