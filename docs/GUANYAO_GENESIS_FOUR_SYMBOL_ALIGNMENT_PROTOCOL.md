# GUANYAO Genesis Four Symbol Alignment Protocol V1.0

## Purpose

P105 establishes the first four-symbol alignment expression after the birth
mansion seed has been claimed. It is a semantic projection only:

```text
Birth Mansion Ignition Projection
+ Formal Four Symbol Result Reference
↓
Genesis Four Symbol Alignment Projection
↓
Life Presence Projection
↓
Isolated Prototype Renderer
```

## Meaning of Four Symbol

Four Symbol is the morphological field in which a personal star-mansion life
becomes perceptible. It is not a summoned animal, an animal asset, a pet, a
character, or a visual identity chosen by the renderer.

> 四象不是召唤出来的兽，而是本命星宿生命被宇宙看见后的形态场。

The alignment layer therefore expresses spatial bias, boundary behaviour,
directional flow, posture tendency, and envelope scale. It never emits animal
geometry or names a creature.

The projection reaches `FIELD_ALIGNED` only after the seed claim is complete;
its `morphologicalFieldExpression` carries the generic field parameters and
always declares `noAnimalGeometry`.

## Source and boundary

The formal four-symbol engine result remains an independent source. P105 does
not calculate four symbols, select a symbol, calculate MotherCode, or create a
Personal Star Beast. It consumes only an opaque formal result reference after
the P104 seed-claimed state.

The renderer receives only expression parameters and references. It must not
read mansion facts, four-symbol facts, MotherCode, LifeArchetype, or user data.

## Field modes

The mapping is generic morphology, not animal modelling:

- `EXTENDING_ARC`: spatial extension and outward continuity.
- `RISING_FLOW`: upward movement and opening flow.
- `CONVERGING_BOUNDARY`: contraction and gathered edges.
- `ENCLOSING_DEPTH`: containment, depth, and protected interior.

These modes change the organization of the life structure without drawing a
white tiger, dragon, bird, tortoise, or any other animal.

## Human acceptance question

The P105 question is not “which beast was summoned?” It is:

> 本命星宿生命被宇宙看见后，形态场是否开始形成？

P105 remains `ISOLATED_PROTOTYPE_ONLY`; it does not enter Production, UI,
Storage, Gravity, Crystal, Memory, or Runtime.
