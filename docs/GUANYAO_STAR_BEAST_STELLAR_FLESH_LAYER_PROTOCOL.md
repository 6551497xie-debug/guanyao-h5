# GUANYAO Star Beast Stellar Flesh Layer Protocol

协议编号：`RC-STAR-BEAST-STELLAR-FLESH-LAYER-P87`

P87 在 `COSMIC_CONSCIOUSNESS` 与 `WHITE_TIGER_REVEAL` 之间建立 Stellar Flesh 生命质感语义层。它不新增 Genesis 阶段，也不生成动物皮肤、纹理贴图或填充颜色。

## 1. 定义与数据链

Stellar Flesh 是星辰秩序聚合后形成的半透明生命场。

`CosmicConsciousnessState → StellarFleshState → Isolated Prototype Renderer`

`StellarFleshState` 只保存密度、内部生命场、边界存在场、意识与 Presence 的表达引用，不复制坐标、粒子数量、颜色、透明度或动画参数等视觉事实。

## 2. 视觉原则

- 星辰肉身不是填满轮廓：内部保持稀薄星尘场，通过层次、深浅与缓慢流动形成生命质感。
- 空间厚度不是 3D 模型：只允许半透明分层、局部光场和密度变化。
- 艮不是更多光：艮之白虎表现内部沉积、缓慢流动与稳定边界，禁止爆发和扩张。
- Presence 不是动作：最终状态保持静默在场，不增加动物表演。

## 3. 五阶段映射

1. `DORMANT → DORMANT`：没有肉身生命场。
2. `ORDERING → SEEDING`：只建立密度来源，不显示兽形。
3. `EMERGING → THIN_FIELD`：生命结构内部出现稀薄、半透明分层。
4. `AWAKENING → SETTLING_FIELD`：局部密度向内部沉积，边界保持稳定。
5. `QUIET_PRESENCE → STABLE_LIVING_FIELD`：半透明生命场稳定存在，保留稀疏流动。

## 4. 工程边界

Mapping 的唯一来源是 `CosmicConsciousnessState`，不得反推 TwentyEightMansion、FourSymbol、LifeArchetype 或生命来源。它不得修改 P84 Genesis Asset、Genesis State、Life State 或 Renderer Contract，不得创建 Memory、Growth、Crystal 或用户画像。

Canvas 只消费 Stellar Flesh 表达状态。Scope 固定为 `ISOLATED_PROTOTYPE_ONLY`；禁止接入 Launch、Gravity、Dynamics、Crystal UI、Storage 或正式用户流程。继续禁止 WebGL、Shader、Three.js、3D 模型、毛发、写实动物与爆发粒子。
